# Few-shot Example Techniques



## What is Few-shot?

**Few-shot (Few-shot Learning)** means providing a few examples in the prompt, allowing AI to learn the expected output format or behavior pattern through those examples.

> Just like teaching a child to recognize characters: you show them a few pictures of "apple", and they can identify other apples.

## Why is Few-shot Effective?

### 1. Clarify Output Format

AI learns what kind of output you want through examples.

```markdown
❌ Without examples:
"Convert the following information to JSON"

Result: AI might output JSON in various formats

✅ With examples:
"Convert the following information to JSON format:

Example 1:
Input: Zhang San, 25 years old, Beijing
Output: {\"name\": \"Zhang San\", \"age\": 25, \"city\": \"Beijing\"}

Example 2:
Input: Li Si, 30 years old, Shanghai
Output: {\"name\": \"Li Si\", \"age\": 30, \"city\": \"Shanghai\"}

Now convert:
Input: Wang Wu, 28 years old, Guangzhou
Output: "

Result: AI will output in the example format
```

### 2. Specify Style Requirements

Tell AI the style you want through examples.

```markdown
❌ Without examples:
"Write code comments in a concise style"

Result: AI's understanding of "concise" might differ from yours

✅ With examples:
"Please add concise comments to the code, style as follows:

Example:
```javascript
// Calculate user age
function calculateAge(birthDate) {
  return new Date().getFullYear() - birthDate.getFullYear();
}
```

Note:
- Comments should be brief, no more than 10 characters
- Explain "what to do", not "how to do"
- Start with verbs"

Result: AI will write comments in the example style
```

### 3. Handle Complex Tasks

For complex tasks, examples help AI understand the task logic.

```markdown
❌ Without examples:
"Convert natural language to SQL"

Result: AI might generate incorrect SQL

✅ With examples:
"Convert natural language to SQL queries:

Example 1:
Natural language: Query all users older than 25
SQL: SELECT * FROM users WHERE age > 25;

Example 2:
Natural language: Query average salary for each department
SQL: SELECT department, AVG(salary) FROM employees GROUP BY department;

Now convert:
Natural language: Query the number of users registered in the last 7 days
SQL: "

Result: AI can generate correct SQL
```

## Few-shot Best Practices

### 1. Number of Examples

- **Simple tasks**: 1-2 examples are enough
- **Complex tasks**: 3-5 examples
- **Don't overdo it**: More than 5 might waste context

### 2. Example Quality

```markdown
✅ Good examples:
- Correct format
- Clear logic
- Cover common cases
- Include boundary cases

❌ Bad examples:
- Have errors
- Inconsistent format
- Too simple
- Ambiguous
```

### 3. Example Diversity

Provide different types of examples to help AI understand various situations.

```markdown
Task: Identify user intent

Example 1 (Query):
User: "What's the weather in Beijing today?"
Intent: Weather query

Example 2 (Order):
User: "I want to book a flight to Shanghai tomorrow"
Intent: Book flight

Example 3 (Complaint):
User: "Your service is terrible, I want a refund!"
Intent: Complaint/refund

Example 4 (Consultation):
User: "What privileges do members have?"
Intent: Information consultation
```

### 4. Consistent Example Format

All examples should use the same format for easy AI learning.

```markdown
✅ Consistent format:

Example 1:
Input: ...
Output: ...

Example 2:
Input: ...
Output: ...

❌ Inconsistent format:

Example 1:
Input: ...
Result: ...

Example 2:
User said: ...
AI answered: ...
```

## Advanced Techniques

### Technique 1: Contrastive Examples

Tell AI what you want and what you don't want through positive and negative examples.

```markdown
Task: Generate concise code comments

✅ Good comment:
```javascript
// Calculate order total
function calculateTotal(order) {
  return order.items.reduce((sum, item) => sum + item.price, 0);
}
```

❌ Bad comment:
```javascript
// This function receives an order object, then iterates through all items in the order,
// adds up the price of each item, and finally returns the sum
function calculateTotal(order) {
  return order.items.reduce((sum, item) => sum + item.price, 0);
}
```

```

### Technique 2: Step-by-step Examples

For complex tasks, show the thinking process.

```markdown
Task: Solve math problems

Example:

Problem: A rectangle has length 8cm and width 5cm, find the area.

Thinking process:
1. Rectangle area formula: Area = Length × Width
2. Substitute values: Area = 8cm × 5cm
3. Calculate result: Area = 40cm²

Answer: 40cm²

Now solve:
Problem: A circle has radius 3cm, find the area (use π = 3.14)
```

Combine Few-shot with clear instructions.

Requirements:

Examples:

## Common Mistakes

### Mistake 1: Too Few Examples

```markdown
❌ Only one example
AI might not fully understand the pattern

✅ 2-5 examples
Cover different situations
```

### Mistake 2: Errors in Examples

```markdown
❌ AI will learn from errors in examples

Example:
Input: 2 + 2
Output: 5  ← Wrong!

Result: AI will also output wrong answers
```

### Mistake 3: Examples Don't Match Task

```markdown
❌ Examples and actual task differ too much

Example is: Text classification
Actual task: Code generation

Result: AI cannot understand
```
