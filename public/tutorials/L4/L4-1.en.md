# Structured Prompts



## Why Do We Need Structured Prompts?

Imagine how you would tell someone to do something:

> ❌ **Bad approach**:
> "Go make something"

> **Result**: The person has no idea what to do at all

---

> ✅ **Good approach**:
>
> "You are a chef (role),
>  now need to prepare dinner for 5 people (context),
>  please make three dishes: one meat dish, one vegetable dish, one soup (task),
>  requirements: not spicy, completed within 30 minutes, budget 100 yuan (constraints)"
>
> **Result**: The person knows exactly what to do

**Structured prompts are about making AI understand clearly what you want.**

## Core Framework of Structured Prompts

### RCTC Framework

| Element | English | Description | Example |
|---------|---------|-------------|---------|
| **R**ole | Role | What role AI should play | You are a senior frontend engineer |
| **C**ontext | Context | Relevant background information | We are developing an e-commerce website |
| **T**ask | Task | What task to complete | Please implement shopping cart functionality |
| **C**onstraints | Constraints | Any limitations | Use React, support TypeScript |

### Extended Framework: RCTC + E

Add **E**xamples on top of RCTC:

| Element | Description |
|---------|-------------|
| **R**ole | Role |
| **C**ontext | Context |
| **T**ask | Task |
| **C**onstraints | Constraints |
| **E**xamples | Examples |

## Detailed Explanation of Each Element

### 1. Role

**Purpose**: Have AI answer questions from a specific perspective, yielding more professional output.

**Examples**:

> ❌ **Without role**:
> "Explain React"

---

> ✅ **With role**:
> "You are a frontend architect with 10 years of experience,
>  please explain React's core concepts to a junior developer"
>
> **Result**: Explanation is more in-depth, professional, and suitable for beginners

**Common roles**:
- Technical experts: Senior engineers, architects, technical directors
- Educators: Teachers, mentors, trainers
- Business experts: Product managers, designers, operations
- Reviewers: Code reviewers, QA engineers

### 2. Context

**Purpose**: Provide necessary background so AI understands the context of the question.

**Examples**:

> ❌ **Without context**:
> "Optimize this code"

---

> ✅ **With context**:
> "We are developing a high-concurrency e-commerce system,
>  this code is the product search API,
>  currently response time exceeds 2 seconds, needs to be optimized to within 200ms"
>
> **Result**: AI will provide targeted optimization suggestions

**Background information includes**:
- Project type and scale
- Tech stack and versions
- Target users and scenarios
- Current problems and pain points
- Already attempted solutions

### 3. Task

**Purpose**: Clearly tell AI what to do.

**Examples**:
```markdown
❌ Vague task:
"Process this data"

✅ Clear task:
"Please analyze this sales data:
 1. Calculate total sales for each month
 2. Find the product with highest sales
 3. Analyze sales trends
 4. Display results with charts"

Result: AI knows exactly what to do
```

**Task description tips**:
- Start with verbs: Implement, analyze, optimize, create...
- Decompose into specific steps
- Clarify output format

### 4. Constraints

**Purpose**: Limit AI's output scope to avoid results that don't meet requirements.

**Examples**:
```markdown
❌ Without constraints:
"Write a login feature"

✅ With constraints:
"Write a login feature with requirements:
 - Use JWT authentication
 - Password encrypted with bcrypt
 - Support Remember Me (7 days)
 - Response time < 100ms
 - Code with complete comments"

Result: AI will generate code following the constraints
```

**Common constraint types**:
- Technical constraints: Language, framework, library versions
- Performance constraints: Response time, memory usage
- Security constraints: Encryption methods, permission control
- Format constraints: Code style, documentation format
- Time constraints: Completion time, deadlines

### 5. Examples

**Purpose**: Explain expected output format and quality through examples.

**Examples**:
```markdown
Without examples:
"Classify the following text"

With examples:
"Please classify the following text as: Positive, Negative, Neutral

Example:
Text: 'This product is amazing!'
Classification: Positive

Text: 'Quality is very poor, not recommended'
Classification: Negative

Text: 'The weather is nice today'
Classification: Neutral

Now classify:
Text: 'Shipping was fast, packaging was intact'
Classification: ?"

Result: AI knows the classification criteria and format
```

## Complete Examples

### Example 1: Code Review

```markdown
## Role
You are a senior frontend code reviewer with 10 years of React development experience.

## Context
We are developing an enterprise-level admin management system using React 18 + TypeScript.
This is the user management page code, need to ensure code quality and performance.

## Task
Please review the following code and identify potential issues:

```tsx
import React, { useState, useEffect } from 'react';

function UserList() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch('/api/users')
      .then(res => res.json())
      .then(data => setUsers(data));
  });

  return (
    <div>
      {users.map(user => (
        <div key={user.id}>{user.name}</div>
      ))}
    </div>
  );
}
```

## Constraints
1. Focus on: Performance issues, memory leaks, type safety
2. For each issue provide: Severity, specific location, fix suggestions
3. Provide optimized complete code
```

### Example 2: Feature Implementation

```markdown
## Role
You are a full-stack engineer proficient in Node.js and PostgreSQL.

## Context
We are developing a blog system and need to implement article comment functionality.
Tech stack: Node.js + Express + PostgreSQL + TypeScript

## Task
Please implement comment functionality API including:
1. Post comment
2. Get comment list (with pagination)
3. Delete comment (author or admin only)

## Constraints
- Use RESTful API design
- Input validation required (use zod)
- Error handling required
- Code with complete comments
- Include unit tests

## Examples
Input validation example:
```typescript
const createCommentSchema = z.object({
  postId: z.string().uuid(),
  content: z.string().min(1).max(1000),
  parentId: z.string().uuid().optional()
});
```

### Example 3: Documentation Writing

#### Role
You are a technical documentation engineer proficient in writing clear and easy-to-understand technical documentation.

#### Context
We developed a React component library and need to write documentation for the Button component.
Target audience: Junior to intermediate frontend developers

#### Task
Please write documentation for the Button component including:
1. Component introduction
2. Props explanation
3. Usage examples
4. Common questions

#### Constraints
- Use Markdown format
- Include code examples
- Concise and clear language
- Suitable for quick start

#### Examples
Props table format:
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| size | 'sm' \| 'md' \| 'lg' | 'md' | Button size |

## Common Mistakes

### Mistake 1: Missing Role

> ❌ "Explain what a closure is"
>
> ✅ "You are a JavaScript expert, please explain what a closure is to beginners using real-life examples"

### Mistake 2: Insufficient Context

> ❌ "Optimize this code"
>
> ✅ "This code is for the user list page, currently has 10000+ users and page loads slowly, please optimize performance"

### Mistake 3: Vague Task

> ❌ "Handle this requirement"
>
> ✅ "Please analyze this requirement and output:
>      1. Feature module division
>      2. Data model design
>      3. Interface definition"

### Mistake 4: Unclear Constraints

> ❌ "Write a login feature"
>
> ✅ "Write a login feature with requirements:
>      - Use email + password
>      - JWT Token authentication
>      - Token valid for 24 hours
>      - User-friendly error messages"
