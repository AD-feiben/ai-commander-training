# Iterative Prompt Optimization



## What is Iterative Prompting?

**Iterative prompting** means through multiple rounds of dialogue, gradually optimizing AI's output. Just like polishing a stone, each round makes the result a little better.

> Don't expect perfection in one go, iteration is the norm.

## Why Do We Need Iteration?

### Limitations of AI Single Generation

- **Understanding deviation**: AI might misunderstand your intent
- **Missing details**: Complex tasks easily overlook details
- **Uneven quality**: Some parts good, some parts bad
- **Boundary cases**: Easily overlooked boundary conditions

### Benefits of Iteration

- **Gradual improvement**: From 60 points to 90 points
- **Timely error correction**: Discover and fix problems immediately
- **Deep optimization**: Improve specific aspects
- **Meet expectations**: Final result meets your requirements

## Basic Iteration Process

```
Round 1: Generate initial version
    ↓
Round 2: Point out problems → AI corrects
    ↓
Round 3: Optimize details → AI improves
    ↓
Round 4: Final confirmation → Complete
```

## Practical Examples

### Example 1: Code Generation Iteration

**Round 1: Generate initial version**

```
You: Help me write an email validation function

AI:
```javascript
function validateEmail(email) {
  return email.includes('@');
}
```

**Round 2: Point out problems**

```
You: This validation is too simple, need stricter validation:
1. Validate email format (regular expression)
2. Validate length (max 254 characters)
3. Validate domain part

AI:
```javascript
function validateEmail(email) {
  if (!email || email.length > 254) {
    return false;
  }
  const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return regex.test(email);
}
```

**Round 3: Optimize details**

```
You: Good! Now optimize further:
1. Add JSDoc comments
2. Return specific error messages, not just true/false
3. Handle empty strings and null

AI:
```javascript
/**
 * Validate email address
 * @param email - Email to validate
 * @returns Validation result object
 */
function validateEmail(email) {
  // Check for null values
  if (!email) {
    return { valid: false, error: 'Email cannot be empty' };
  }

  // Check type
  if (typeof email !== 'string') {
    return { valid: false, error: 'Email must be a string' };
  }

  // Check length
  if (email.length > 254) {
    return { valid: false, error: 'Email length cannot exceed 254 characters' };
  }

  // Check format
  const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  if (!regex.test(email)) {
    return { valid: false, error: 'Email format is incorrect' };
  }

  return { valid: true, error: null };
}
```

**Round 4: Final optimization**

```
You: Final optimization:
1. Use TypeScript
2. Add unit tests

AI:
```typescript
interface ValidationResult {
  valid: boolean;
  error: string | null;
}

/**
 * Validate email address
 * @param email - Email to validate
 * @returns Validation result object
 */
function validateEmail(email: unknown): ValidationResult {
  if (!email) {
    return { valid: false, error: 'Email cannot be empty' };
  }

  if (typeof email !== 'string') {
    return { valid: false, error: 'Email must be a string' };
  }

  if (email.length > 254) {
    return { valid: false, error: 'Email length cannot exceed 254 characters' };
  }

  const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  if (!regex.test(email)) {
    return { valid: false, error: 'Email format is incorrect' };
  }

  return { valid: true, error: null };
}

// Unit tests
describe('validateEmail', () => {
  test('valid email', () => {
    expect(validateEmail('test@example.com')).toEqual({
      valid: true,
      error: null
    });
  });

  test('empty email', () => {
    expect(validateEmail('')).toEqual({
      valid: false,
      error: 'Email cannot be empty'
    });
  });

  test('invalid format', () => {
    expect(validateEmail('invalid-email')).toEqual({
      valid: false,
      error: 'Email format is incorrect'
    });
  });
});
```

## Iteration Techniques

### Technique 1: Specific Feedback

❌ Bad feedback:
```
"This code is not good"
```

✅ Good feedback:
```
"This code has problems handling edge cases:
- Will error when input is null
- Doesn't validate parameter types
- Should add error handling"
```

### Technique 2: Change One Thing at a Time

❌ Asking too much at once:
```
"Please modify: add types, optimize performance, add comments,
fix bugs, add tests, improve naming..."
```

✅ One focus at a time:
```
Round 1: "Please add TypeScript types"
Round 2: "Please fix edge case handling"
Round 3: "Please add unit tests"
```

### Technique 3: Be Clear About Requirements

❌ Saying what you don't want:
```
"Don't make it too complicated"
```

✅ Saying what you want:
```
"Please keep it simple, only keep core functions,
remove unnecessary abstractions"
```

### Technique 4: Provide References

```
"Please modify referring to this style:
[Paste reference code]

Please rewrite your code in this format"
```

## Iteration Patterns

### Pattern 1: Error Correction Iteration

Find problem → Point out problem → AI corrects

```
You: This code has a bug, it errors when array is empty
AI: Fixed, added empty array check
```

### Pattern 2: Enhancement Iteration

Basic version → Supplement details → Add features

```
You: Please add error handling
You: Please add logging
You: Please add performance monitoring
```

### Pattern 3: Optimization Iteration

Can run → Code clean → Performance optimized

```
You: Please refactor code, improve readability
You: Please optimize algorithm, improve performance
```

## Common Mistakes

### Mistake 1: Expecting Perfection in One Go

```
❌ Asking for everything in first round
Result: AI cannot process everything at once, quality decreases

✅ Gradually improve over multiple rounds
Result: Each round has improvement, final quality is high
```

### Mistake 2: Vague Feedback

```
❌ "Doesn't feel right"
Result: AI doesn't know what to change

✅ "Line 3 logic has a problem, should..."
Result: AI can modify accurately
```

### Mistake 3: Frequently Changing Requirements

```
❌ Changing requirement direction every round
Result: Never completed

✅ Keep requirements stable, gradually improve
Result: Efficient completion
```

## Iteration Checklist

Check after each iteration:

- [ ] Did it solve the problem from the previous round?
- [ ] Did it introduce new problems?
- [ ] Is it closer to the expected result?
- [ ] Is there still room for improvement?
