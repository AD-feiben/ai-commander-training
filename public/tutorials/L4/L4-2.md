# Few-shot 示例技巧



## 什么是 Few-shot？

**Few-shot（少样本学习）** 是指在提示词中给出几个示例，让 AI 通过示例学习你期望的输出格式或行为模式。

> 就像教小孩认字：你给他看几个"苹果"的图片，他就能识别其他苹果了。

## 为什么 Few-shot 有效？

### 1. 明确输出格式

AI 通过示例知道你想要什么样的输出。

```markdown
❌ 没有示例：
"将以下信息转换为 JSON"

结果：AI 可能输出各种格式的 JSON

✅ 有示例：
"将以下信息转换为 JSON 格式：

示例 1：
输入：张三，25岁，北京
输出：{\"name\": \"张三\", \"age\": 25, \"city\": \"北京\"}

示例 2：
输入：李四，30岁，上海
输出：{\"name\": \"李四\", \"age\": 30, \"city\": \"上海\"}

现在转换：
输入：王五，28岁，广州
输出："

结果：AI 会按照示例格式输出
```

### 2. 指定风格要求

通过示例告诉 AI 你想要的风格。

```markdown
❌ 没有示例：
"用简洁的风格写代码注释"

结果：AI 对"简洁"的理解可能和你不同

✅ 有示例：
"请为代码添加简洁的注释，风格如下：

示例：
```javascript
// 计算用户年龄
function calculateAge(birthDate) {
  return new Date().getFullYear() - birthDate.getFullYear();
}
```

注意：
- 注释简短，不超过 10 个字
- 说明"做什么"，不说明"怎么做"
- 使用动词开头"

结果：AI 会按照示例风格写注释
```

### 3. 处理复杂任务

对于复杂任务，示例可以帮助 AI 理解任务逻辑。

```markdown
❌ 没有示例：
"将自然语言转换为 SQL"

结果：AI 可能生成错误的 SQL

✅ 有示例：
"将自然语言转换为 SQL 查询：

示例 1：
自然语言：查询所有年龄大于 25 岁的用户
SQL：SELECT * FROM users WHERE age > 25;

示例 2：
自然语言：查询每个部门的平均工资
SQL：SELECT department, AVG(salary) FROM employees GROUP BY department;

现在转换：
自然语言：查询最近 7 天内注册的用户数量
SQL："

结果：AI 能生成正确的 SQL
```

## Few-shot 的最佳实践

### 1. 示例数量

- **简单任务**：1-2 个示例足够
- **复杂任务**：3-5 个示例
- **不要太多**：超过 5 个可能浪费上下文

### 2. 示例质量

```markdown
✅ 好的示例：
- 格式正确
- 逻辑清晰
- 覆盖常见情况
- 包含边界情况

❌ 不好的示例：
- 有错误
- 格式不一致
- 过于简单
- 有歧义
```

### 3. 示例多样性

提供不同类型的示例，帮助 AI 理解各种情况。

```markdown
任务：识别用户意图

示例 1（查询）：
用户："今天北京天气怎么样？"
意图：查询天气

示例 2（订购）：
用户："我想订一张明天去上海的机票"
意图：订购机票

示例 3（投诉）：
用户："你们的服务太差了，我要退款！"
意图：投诉退款

示例 4（咨询）：
用户："会员有什么特权？"
意图：咨询信息
```

### 4. 示例格式统一

所有示例使用相同的格式，便于 AI 学习。

```markdown
✅ 格式统一：

示例 1：
输入：...
输出：...

示例 2：
输入：...
输出：...

❌ 格式不统一：

示例 1：
输入：...
结果：...

示例 2：
用户说：...
AI 回答：...
```

## 进阶技巧

### 技巧 1：对比示例

通过正反示例告诉 AI 什么是要的，什么是不要的。

```markdown
任务：生成简洁的代码注释

✅ 好的注释：
```javascript
// 计算订单总价
function calculateTotal(order) {
  return order.items.reduce((sum, item) => sum + item.price, 0);
}
```

❌ 不好的注释：
```javascript
// 这个函数接收一个订单对象，然后遍历订单中的所有商品，
// 把每个商品的价格加起来，最后返回总和
function calculateTotal(order) {
  return order.items.reduce((sum, item) => sum + item.price, 0);
}
```

### 技巧 2：逐步示例

对于复杂任务，展示思考过程。

```markdown
任务：解决数学问题

示例：

问题：一个长方形的长是 8cm，宽是 5cm，求面积。

思考过程：
1. 长方形面积公式：面积 = 长 × 宽
2. 代入数值：面积 = 8cm × 5cm
3. 计算结果：面积 = 40cm²

答案：40cm²

现在解决：
问题：一个圆的半径是 3cm，求面积（π取 3.14）
```

### 技巧 3：少样本 + 指令结合

结合 Few-shot 和明确的指令。

```markdown
任务：将中文翻译成英文

要求：
1. 保持原意
2. 语言自然流畅

示例：

中文：今天天气真好。
英文：What a nice day today!

## 常见错误

### 错误 1：示例太少

```markdown
❌ 只有一个示例
AI 可能无法充分理解模式

✅ 2-5 个示例
覆盖不同情况
```

### 错误 2：示例有错误

```markdown
❌ 示例中的错误会被 AI 学习

示例：
输入：2 + 2
输出：5  ← 错误！

结果：AI 也会输出错误答案
```

### 错误 3：示例与任务不匹配

```markdown
❌ 示例和实际任务差异太大

示例是：文本分类
实际任务：代码生成

结果：AI 无法理解
```
