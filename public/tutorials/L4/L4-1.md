# 结构化提示词

## 什么是结构化提示词

结构化提示词是一种组织指令的方式，让 AI 能够更准确地理解你的需求。

## 核心要素：RCTCs

结构化提示词包含四个核心要素：

| 要素 | 英文 | 说明 |
|------|------|------|
| 角色 | Role | AI 扮演什么角色 |
| 背景 | Context | 相关的背景信息 |
| 任务 | Task | 要完成什么任务 |
| 约束 | Constraint | 有什么限制条件 |

## 示例

### ❌ 不好

```
写一个函数验证密码
```

### ✅ 好

```
你是一个专业的安全工程师（角色）

需要为一个用户系统编写密码验证函数（背景）

请用 TypeScript 写一个密码验证函数，满足以下要求（任务）：
1. 长度至少 8 个字符
2. 包含大写字母
3. 包含小写字母
4. 包含数字
5. 包含特殊字符

请返回 boolean，true 表示验证通过（约束）
```

## 模板

```markdown
## 角色
[你是一个...]

## 背景
[相关上下文...]

## 任务
[要完成什么...]

## 约束
1. ...
2. ...
3. ...
```

## 实际应用

```markdown
## 角色
你是一个资深的前端架构师，擅长 React 性能优化。

## 背景
我们正在开发一个大型 SaaS 应用，有 100+ 个页面。

## 任务
请帮我审查以下 React 组件的代码，找出性能问题：

[代码...]

## 约束
1. 重点关注渲染性能
2. 识别内存泄漏风险
3. 提出具体的优化建议
```
