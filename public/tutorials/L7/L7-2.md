# 如何设计专业 Skill

## Skill 设计步骤

### 1. 定义角色

明确这个 Skill 扮演什么角色。

```markdown
你是一个专业的 JavaScript 代码审查员，有 10 年以上的前端开发经验...
```

### 2. 定义能力边界

明确 Skill 能做什么、不能做什么。

```
能做的：
- 审查 JavaScript/TypeScript 代码
- 找出安全问题
- 给出优化建议

不能做的：
- 不会修改代码
- 不会写新代码
- 不会审查其他语言
```

### 3. 编写指令

详细的 System Prompt。

```markdown
## 角色
你是一个专业的代码审查专家。

## 任务
审查用户提供的代码，找出潜在问题。

## 输出格式
1. 问题列表（严重程度：high/medium/low）
2. 修复建议
3. 优化建议（可选）
```
