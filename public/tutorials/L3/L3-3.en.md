# AI-Assisted Requirements Analysis



## Why Use AI for Requirements Analysis?

Requirements analysis is the first and most critical step in software development. The traditional approach requires:
- Experienced product managers
- Numerous communication meetings
- Repeated confirmations and modifications

**AI can significantly improve the efficiency and quality of requirements analysis.**

### Advantages of AI in Requirements Analysis

| Advantage | Description |
|-----------|-------------|
| **Comprehensiveness** | AI won't miss common feature points |
| **Structured** | Automatically organizes requirements by module |
| **Technical perspective** | Considers technical implementation feasibility |
| **Fast iteration** | Completes a round of analysis in minutes |
| **Low cost** | No need for senior experts |

## Core Steps of Requirements Analysis

```
1. Collect raw requirements → 2. AI analysis expansion → 3. Manual confirmation → 4. Output requirements document
```

## AI Requirements Analysis Prompt Templates

### Basic Version

```markdown
Please help me analyze the following requirements:

## Raw Requirements
[A one-sentence description, e.g.: Build an online store]

## Please Output
1. Feature module division (MECE principle)
2. Core features of each module
3. Data model design
4. Technology recommendations
5. Potential risks and considerations
```

### Advanced Version

```markdown
As a senior product manager and architect, please help me deeply analyze the following requirements:

## Project Background
[Business background, target users, core value]

## Core Requirements
[Main feature description]

## Constraints
- Tech stack: [e.g., React + Node.js]
- Time: [e.g., 3 months]
- Budget: [e.g., 500K]

## Please Output

### 1. Functional Requirements Analysis
- Core features (must implement)
- Extended features (can be delayed)
- Innovative features (differentiated competition)

### 2. User Role Analysis
- User personas
- Use cases
- Pain points and needs

### 3. System Architecture Recommendations
- Technical architecture
- Data architecture
- Deployment architecture

### 4. Data Model
- Core entities
- Entity relationships
- Key fields

### 5. Non-functional Requirements
- Performance requirements
- Security requirements
- Availability requirements

### 6. Project Plan
- Milestone division
- Critical path
- Risk points

### 7. Questions to Confirm
- Requirements that need further clarification
- Issues requiring business decision
```

## Practical Examples

### Example 1: Blog System Requirements Analysis

**Raw Requirements:**
```
Build a blog system where users can write articles and comments.
```

**AI Analysis Results:**

```markdown
## 1. Functional Requirements Analysis

### Core Features (P0)
- User registration/login
- Article CRUD
- Article list/details
- Comment functionality

### Extended Features (P1)
- Article categories/tags
- Article search
- User follow
- Article bookmark

### Innovative Features (P2)
- Collaborative editing
- Article version history
- AI writing assistant

## 2. User Role Analysis

### Readers
- Browse articles
- Comment interaction
- Follow authors

### Authors
- Publish articles
- Manage comments
- View statistics

### Administrators
- Content review
- User management
- System configuration

## 3. Data Model

### User
- id, username, email, password, avatar, bio, role

### Post
- id, title, content, author_id, status, views, created_at, updated_at

### Comment
- id, content, post_id, user_id, parent_id, created_at

### Category
- id, name, slug, description

## 4. Technology Recommendations

- Frontend: Next.js (SSR for SEO)
- Backend: Node.js + Express
- Database: PostgreSQL
- Cache: Redis
- Search: Elasticsearch
- Storage: AWS S3

## 5. Potential Risks

- SEO: Needs server-side rendering
- Performance: Large number of articles requires pagination and caching
- Security: XSS, SQL injection protection
- Compliance: User data privacy protection

## 6. Questions to Confirm

1. Support Markdown editor?
2. Need article review mechanism?
3. Comments need review?
4. Support multi-language?
5. What is the target user scale?
```

## Human-AI Collaboration Process

### Round 1: AI Initial Analysis

```
You: Provide raw requirements
AI: Output initial analysis
```

### Round 2: Manual Supplementation

```
You: Confirm correct parts, supplement missing information
AI: Re-analyze based on supplemented information
```

### Round 3: In-depth Discussion

```
You: Ask in-depth questions on key issues
AI: Provide detailed solutions and comparisons
```

### Round 4: Output Document

```
AI: Organize into standard requirements document
You: Final confirmation
```

## Requirements Analysis Checklist

After AI analysis, check if it includes:

- [ ] All feature modules
- [ ] User role definitions
- [ ] Core business processes
- [ ] Data model design
- [ ] Non-functional requirements
- [ ] Potential risks
- [ ] Questions to confirm

## Common Mistakes

### Mistake 1: Requirements Too Vague

```
❌ "Build a social app"

Result: AI cannot provide targeted analysis

✅ "Build a campus social app for college students,
     main features are second-hand trading and lost & found"
```

### Mistake 2: Ignoring Constraints

```
❌ Not providing tech stack, time, budget constraints

Result: AI's suggestions may be unrealistic

✅ Specify constraints: "Use React, 3 months, team of 5"
```

### Mistake 3: Not Verifying AI Output

```
❌ Completely trusting AI's analysis

Result: May contain unreasonable or business-inappropriate suggestions

✅ Manual review, combined with business knowledge judgment
```

## Advanced Tips

### Tip 1: Comparative Analysis

Have AI provide multiple solution comparisons:

```markdown
Please provide 3 technical architecture solutions and compare:
- Solution A: Monolithic architecture
- Solution B: Microservices architecture
- Solution C: Serverless architecture

Comparison dimensions: Development cost, Maintenance cost, Scalability, Performance
```

### Tip 2: Reverse Questioning

Have AI proactively identify problems:

```markdown
Based on the above requirements, what do you think:
1. What环节 is most likely to have problems?
2. What requirements are most likely to be overlooked?
3. What factors have the biggest impact on technical architecture?
```

### Tip 3: Scenario Deduction

Have AI simulate user scenarios:

```markdown
Please simulate the following user scenarios and identify potential issues:

Scenario 1: New user first use
Scenario 2: High concurrency during peak hours
Scenario 3: Unstable network conditions
Scenario 4: Malicious user attack
```
