# 规范分解：从大规范到小规范

## 大规范 → 小规范

面对复杂功能，我们需要把大规范分解成小规范，就像把大任务分解成小任务一样。

## 分解的层次

### 第一层：项目规范

```markdown
# 电商网站项目规范

## 项目目标
构建一个完整的电商网站

## 功能模块
1. 用户模块
2. 商品模块
3. 购物车模块
4. 订单模块
5. 支付模块
6. 管理后台

## 技术栈
- 前端：React + TypeScript
- 后端：Node.js
- 数据库：PostgreSQL
```

### 第二层：模块规范

```markdown
# 商品模块规范

## 功能列表
1. 商品列表（分页、筛选）
2. 商品详情
3. 商品搜索
4. 商品分类
5. 商品管理（CRUD）

## 数据模型
Product: id, name, description, price, category_id, stock, images
```

### 第三层：功能规范

```markdown
# 商品列表功能规范

## 功能
展示商品列表，支持分页和筛选

## 输入

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | number | 否 | 1 | 页码 |
| pageSize | number | 否 | 20 | 每页数量 |
| categoryId | string | 否 | - | 分类 ID |
| keyword | string | 否 | - | 搜索关键词 |
| sort | string | 否 | newest | 排序方式 |

## 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| list | Product[] | 商品列表 |
| total | number | 总数 |
| page | number | 当前页码 |
| pageSize | number | 每页数量 |
```

## 分解的好处

1. **降低复杂度** - 每次只处理一个小功能
2. **明确依赖** - 知道先做什么、后做什么
3. **便于测试** - 每个小功能可以独立测试
4. **团队协作** - 不同人可以负责不同部分
