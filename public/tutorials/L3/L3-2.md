# MECE 原则

## 什么是 MECE？

**MECE**（发音：me-see）全称 "Mutually Exclusive, Collectively Exhaustive"

- **Mutually Exclusive**：相互独立，不重叠
- **Collectively Exhaustive**：完全穷尽，无遗漏

## MECE 在编程中的应用

### 示例：用户分类

```
❌ 不好（重叠）：
- 普通用户
- VIP 用户
- 管理员
- 超级管理员

问题：VIP 可以是管理员吗？
```

```
✅ 好（不重叠）：
- 普通用户
- VIP 用户
- 管理员

互不重叠，完全覆盖所有角色
```

### 示例：功能模块划分

```
❌ 不好（重叠）：
- 用户管理
- 用户权限
- 角色管理

问题：权限和角色有重叠
```

```
✅ 好（不重叠）：
- 认证模块
- 授权模块
- 用户模块
```

## 如何应用 MECE

### 1. 列出所有元素

先不要管分类，把所有相关元素都列出来。

### 2. 寻找分类维度

找到一个可以覆盖所有元素的维度：
- 按功能划分
- 按层次划分
- 按状态划分
- 按时间顺序划分

### 3. 检查重叠

问自己：这两个类别有重叠吗？

### 4. 检查遗漏

问自己：还有什么是没覆盖到的？

## 常见分类维度

| 维度 | 示例 |
|------|------|
| 功能 | 用户、商品、订单 |
| 层次 | 前端、后端、数据库 |
| 状态 | 进行中、已完成、已取消 |
| 优先级 | 高、中、低 |
| 权限 | 读、写、管理 |
