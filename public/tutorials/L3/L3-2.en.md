# MECE Principle

## What is MECE?

**MECE** (pronounced: me-see) stands for "Mutually Exclusive, Collectively Exhaustive"

- **Mutually Exclusive**: Independent, non-overlapping
- **Collectively Exhaustive**: Completely exhaustive, no omissions

## MECE in Programming

### Example: User Classification

```
❌ Bad (Overlapping):
- Regular users
- VIP users
- Administrators
- Super administrators

Problem: Can VIP be an administrator?
```

```
✅ Good (Non-overlapping):
- Regular users
- VIP users
- Administrators

Mutually exclusive, completely covering all roles
```

### Example: Feature Module Division

```
❌ Bad (Overlapping):
- User management
- User permissions
- Role management

Problem: Permissions and roles overlap
```

```
✅ Good (Non-overlapping):
- Authentication module
- Authorization module
- User module
```

## How to Apply MECE

### 1. List All Elements

Don't worry about classification first, list all related elements.

### 2. Find Classification Dimensions

Find a dimension that can cover all elements:
- Divide by function
- Divide by layer
- Divide by state
- Divide by time sequence

### 3. Check for Overlap

Ask yourself: Do these two categories overlap?

### 4. Check for Omissions

Ask yourself: Is there anything not covered?

## Common Classification Dimensions

| Dimension | Example |
|-----------|---------|
| Function | User, Product, Order |
| Layer | Frontend, Backend, Database |
| State | In progress, Completed, Cancelled |
| Priority | High, Medium, Low |
| Permission | Read, Write, Admin |
