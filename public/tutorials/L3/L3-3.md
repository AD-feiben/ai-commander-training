# 用 AI 进行需求分析

## AI 是需求分析的好帮手

你不仅可以向 AI 询问技术问题，还可以让它帮你进行需求分析。

## 需求分析提示词

```markdown
请帮我分析以下需求，识别功能点和技术要点：

## 需求描述
[你的需求]

## 请输出
1. 功能列表（按模块分组）
2. 数据模型（如果需要）
3. 技术建议
4. 潜在风险
5. 遗漏的问题（请列出你没有明确说明的点）
```

## 示例

```
需求：做一个博客系统
```

让 AI 分析：

```markdown
## 功能模块

### 用户模块
- 注册/登录
- 个人资料管理

### 文章模块
- 创建文章
- 编辑文章
- 删除文章
- 文章列表
- 文章详情
- 分类/标签

### 评论模块
- 发表评论
- 回复评论

## 数据模型

- User: id, username, email, password, avatar
- Post: id, title, content, author_id, created_at
- Comment: id, content, post_id, user_id, parent_id

## 技术建议

- 前端：React + Markdown 编辑器
- 后端：RESTful API
- 数据库：PostgreSQL
- 图片：使用对象存储

## 潜在风险

- 性能：大量文章需要分页
- 安全：XSS 攻击需要防护
- SEO：需要服务端渲染或静态生成
```

## 人机协作

```
你：提供需求
AI：分析功能、识别问题
你：确认和补充
AI：再次分析
...迭代...
```
