# 实战：分解一个完整需求



## 练习目标

本节将带你完整地分解一个实际需求，从原始需求到可执行的任务列表。

通过本练习，你将掌握：
- 需求分析的方法
- 模块分解的技巧
- 任务细化的能力
- 与 AI 协作的流程

## 案例：在线问卷调查系统

### 原始需求

```
做一个在线问卷调查系统：
- 可以创建问卷（选择题、填空题）
- 可以发布问卷链接
- 可以填写问卷
- 可以查看统计结果
- 支持多语言
```

## 第一步：AI 辅助需求分析

首先，让 AI 帮我们分析需求：

```markdown
请帮我分析以下需求：

## 原始需求
做一个在线问卷调查系统，支持创建问卷、发布链接、填写问卷、查看统计、多语言。

## 请输出
1. 功能模块划分（MECE 原则）
2. 每个模块的核心功能
3. 数据模型设计
4. 用户角色分析
5. 潜在的技术难点

```

## 第三步：编写规范

为每个模块编写技术规范。以"创建问卷"为例：

```markdown
# 创建问卷功能规范

## 功能描述
用户可以创建一个新的问卷，添加标题、描述和问题。

## 用户界面

### 页面布局
- 左侧：问题类型选择区
- 中间：问卷编辑区
- 右侧：属性设置区

### 问题类型
1. 单选题（单选按钮）
2. 多选题（复选框）
3. 填空题（文本输入）
4. 评分题（星星评分）

## 数据模型

### Survey
```typescript
interface Survey {
  id: string;
  title: string;
  description?: string;
  creatorId: string;
  status: 'draft' | 'published' | 'closed';
  settings: SurveySettings;
  questions: Question[];
  createdAt: Date;
  updatedAt: Date;
}
```

### Question
```typescript
interface Question {
  id: string;
  type: 'single' | 'multiple' | 'text' | 'rating';
  title: string;
  required: boolean;
  options?: Option[];  // 选择题选项
  maxRating?: number;  // 评分题最大值
  sortOrder: number;
}
```

## 接口定义

### POST /api/surveys
创建问卷

**Request:**
```json
{
  "title": "问卷标题",
  "description": "问卷描述",
  "questions": [
    {
      "type": "single",
      "title": "问题标题",
      "required": true,
      "options": [
        { "label": "选项1", "value": "1" },
        { "label": "选项2", "value": "2" }
      ]
    }
  ],
  "settings": {
    "allowAnonymous": true,
    "deadline": "2025-12-31"
  }
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "id": "survey-uuid",
    "title": "问卷标题",
    "status": "draft",
    "createdAt": "2025-01-15T10:00:00Z"
  }
}
```

## 业务逻辑

1. 验证用户已登录
2. 验证标题不为空（1-200 字符）
3. 验证至少有一个问题
4. 验证每个问题都有标题
5. 选择题必须至少有两个选项
6. 保存到数据库
7. 返回创建的问卷信息

## 边界条件

- 标题为空：提示"请输入问卷标题"
- 标题超过 200 字符：提示"标题过长"
- 没有问题：提示"请至少添加一个问题"
- 问题标题为空：提示"请输入问题标题"
- 选择题选项少于 2 个：提示"请至少添加两个选项"

## 验收标准

- [ ] 可以成功创建问卷
- [ ] 可以添加不同类型的问题
- [ ] 可以设置问题属性
- [ ] 可以设置问卷属性
- [ ] 表单验证正常工作
- [ ] 创建成功后跳转到问卷管理页
```
