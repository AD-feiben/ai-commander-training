# 实战：分解一个完整需求

## 练习目标

本节将带你完整地分解一个实际需求。

## 需求

```
做一个在线问卷调查系统：
- 可以创建问卷（选择题、填空题）
- 可以发布问卷链接
- 可以填写问卷
- 可以查看统计结果
- 支持多语言
```

## 步骤 1：识别主要模块

根据 MECE 原则，分解出主要模块：

```
1. 问卷管理模块 - 创建、编辑、发布、删除
2. 问卷填写模块 - 填写的页面和逻辑
3. 数据收集模块 - 收集和存储回答
4. 统计展示模块 - 展示统计结果
5. 用户模块 - 登录、权限
6. 系统设置 - 多语言、主题
```

## 步骤 2：分解每个模块

### 问卷管理模块

```
1.1 创建问卷
    - 输入问卷标题
    - 添加问题（选择题、填空题）
    - 设置问卷属性（截止时间、是否匿名）
    
1.2 编辑问卷
    - 修改问卷内容
    - 预览问卷
    
1.3 发布问卷
    - 生成问卷链接
    - 设置访问权限
    
1.4 删除问卷
```

### 问卷填写模块

```
2.1 加载问卷
    - 获取问卷内容
    - 验证问卷状态
    
2.2 填写问卷
    - 回答选择题
    - 回答填空题
    - 表单验证
    
2.3 提交问卷
    - 保存回答
    - 显示提交成功
```

## 步骤 3：检查完整性

问自己：
- [ ] 所有功能都覆盖了吗？
- [ ] 有没有重复的模块？
- [ ] 模块之间有依赖吗？

## 总结

分解是复杂任务处理的核心技能：

```
复杂需求 → 模块分解 → 功能分解 → 任务列表
```
