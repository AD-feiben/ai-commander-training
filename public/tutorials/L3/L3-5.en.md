# Practice: Decompose a Complete Requirement

## Exercise Objective

This section will guide you through completely decomposing an actual requirement.

## Requirement

```
Build an online survey system:
- Can create surveys (multiple choice, fill-in-the-blank)
- Can publish survey links
- Can fill out surveys
- Can view statistical results
- Support multiple languages
```

## Step 1: Identify Main Modules

Based on MECE principles, decompose into main modules:

```
1. Survey Management Module - Create, edit, publish, delete
2. Survey Filling Module - Filling page and logic
3. Data Collection Module - Collect and store responses
4. Statistics Display Module - Display statistical results
5. User Module - Login, permissions
6. System Settings - Multi-language, themes
```

## Step 2: Decompose Each Module

### Survey Management Module

```
1.1 Create Survey
    - Enter survey title
    - Add questions (multiple choice, fill-in-the-blank)
    - Set survey properties (deadline, anonymous or not)
    
1.2 Edit Survey
    - Modify survey content
    - Preview survey
    
1.3 Publish Survey
    - Generate survey link
    - Set access permissions
    
1.4 Delete Survey
```

### Survey Filling Module

```
2.1 Load Survey
    - Get survey content
    - Validate survey status
    
2.2 Fill Survey
    - Answer multiple choice questions
    - Answer fill-in-the-blank questions
    - Form validation
    
2.3 Submit Survey
    - Save responses
    - Show submission success
```

## Step 3: Check Completeness

Ask yourself:
- [ ] Are all features covered?
- [ ] Are there any duplicate modules?
- [ ] Are there dependencies between modules?

## Summary

Decomposition is the core skill for handling complex tasks:

```
Complex requirement → Module decomposition → Feature decomposition → Task list
```
