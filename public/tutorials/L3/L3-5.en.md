# Practice: Decomposing a Complete Requirement



## Practice Objectives

This section will guide you through a complete decomposition of an actual requirement, from raw requirements to an executable task list.

Through this exercise, you will master:
- Requirements analysis methods
- Module decomposition techniques
- Task refinement skills
- AI collaboration processes

## Case Study: Online Survey System

### Raw Requirements

```
Build an online survey system:
- Can create surveys (multiple choice, fill-in-the-blank)
- Can publish survey links
- Can fill out surveys
- Can view statistical results
- Support multi-language
```

## Step 1: AI-Assisted Requirements Analysis

First, have AI help us analyze the requirements:

```markdown
Please help me analyze the following requirements:

## Raw Requirements
Build an online survey system supporting survey creation, link publishing, survey filling, statistics viewing, and multi-language.

## Please Output
1. Feature module division (MECE principle)
2. Core features of each module
3. Data model design
4. User role analysis
5. Potential technical challenges
6. Recommended implementation order
```

### AI Analysis Results Example

```markdown

## Step 3: Write Specifications

Write technical specifications for each module. Taking "Create Survey" as an example:

```markdown
# Create Survey Feature Specification

## Feature Description
Users can create a new survey, adding title, description, and questions.

## User Interface

### Page Layout
- Left side: Question type selection area
- Center: Survey editing area
- Right side: Properties settings area

### Question Types
1. Single choice (radio button)
2. Multiple choice (checkbox)
3. Fill-in-the-blank (text input)
4. Rating (star rating)

## Data Model

### Survey
```typescript
interface Survey {
  id: string;
  title: string;
  description?: string;
  creatorId: string;
  status: 'draft' | 'published' | 'closed';
  settings: SurveySettings;
  questions: Question[];
  createdAt: Date;
  updatedAt: Date;
}
```

### Question
```typescript
interface Question {
  id: string;
  type: 'single' | 'multiple' | 'text' | 'rating';
  title: string;
  required: boolean;
  options?: Option[];  // Options for choice questions
  maxRating?: number;  // Max value for rating questions
  sortOrder: number;
}
```

## Interface Definition

### POST /api/surveys
Create survey

**Request:**
```json
{
  "title": "Survey Title",
  "description": "Survey Description",
  "questions": [
    {
      "type": "single",
      "title": "Question Title",
      "required": true,
      "options": [
        { "label": "Option 1", "value": "1" },
        { "label": "Option 2", "value": "2" }
      ]
    }
  ],
  "settings": {
    "allowAnonymous": true,
    "deadline": "2025-12-31"
  }
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "id": "survey-uuid",
    "title": "Survey Title",
    "status": "draft",
    "createdAt": "2025-01-15T10:00:00Z"
  }
}
```

## Business Logic

1. Verify user is logged in
2. Verify title is not empty (1-200 characters)
3. Verify at least one question exists
4. Verify each question has a title
5. Choice questions must have at least two options
6. Save to database
7. Return created survey information

## Boundary Conditions

- Title empty: Prompt "Please enter survey title"
- Title exceeds 200 characters: Prompt "Title too long"
- No questions: Prompt "Please add at least one question"
- Question title empty: Prompt "Please enter question title"
- Choice question options less than 2: Prompt "Please add at least two options"

## Acceptance Criteria

- [ ] Can successfully create a survey
- [ ] Can add different types of questions
- [ ] Can set question properties
- [ ] Can set survey properties
- [ ] Form validation works normally
- [ ] After successful creation, navigate to survey management page
```
