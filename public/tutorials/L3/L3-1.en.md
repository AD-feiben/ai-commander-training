# Why Decompose Tasks

## Task Decomposition is the Foundation of AI Collaboration

When you face a complex task, directly asking AI to execute it often yields unsatisfactory results. The reason is simple: AI needs clear, specific instructions.

## Problems Without Decomposition

```
Requirement: Build an e-commerce website

AI might:
- Generate a simple HTML page
- Not know backend is needed
- Not know database design
- Miss payment, order, and other features
```

This is the result of not decomposing tasks.

## Effect After Decomposition

```
Requirement: Build an e-commerce website

After decomposition:
1. Project Planning
   - Technology selection
   - Feature list
   - Database design
   
2. User Module
   - Registration and login
   - Profile management
   
3. Product Module
   - Product list
   - Product details
   - Search
   
4. Order Module
   - Shopping cart
   - Checkout
   - Payment
   - Order management
   
5. Admin Panel
   - Product management
   - Order management
```

Now let AI implement one by one, efficiency and quality will be greatly improved.

## Task Decomposition Principles

1. **Executable**: Each subtask should be independently executable
2. **Complete**: Cover all requirements
3. **Non-overlapping**: Subtasks should not overlap
4. **Dependencies**: Clarify dependencies between tasks
