## 2. Intelligent Commit Message Generation

### 2.1 Why Commit Messages Matter

```
Good commit message:
feat(auth): Add user login functionality

- Implement JWT Token authentication
- Add login form validation
- Integrate password encryption

Closes #123

Bad commit message:
update code
fix bug
changes
```

### 2.2 AI-Generated Commit Messages

**Prompt Template:**

```markdown
Please generate a commit message following Conventional Commits
spec for the following code changes.

Changes:
```diff
[git diff content]
```

Requirements:
1. Use Conventional Commits format
2. Subject no more than 50 characters
3. Body explains why and what changed
4. Mark BREAKING CHANGE if applicable

Output format:
type(scope): subject

body

footer
```

**Example Output:**

```
feat(api): Add user registration endpoint

- Implement /api/register endpoint
- Add email format validation
- Integrate bcrypt password encryption
- Add duplicate user detection

Closes #45
```

### 2.3 Implementation

```typescript
class CommitMessageGenerator {
  private openai: OpenAI;

  constructor(apiKey: string) {
    this.openai = new OpenAI({ apiKey });
  }

  async generate(diff: string): Promise<string> {
    const prompt = `
Please generate a commit message following Conventional Commits
spec for the following code changes.

Changes:
\`\`\`diff
${diff}
\`\`\`

Requirements:
1. type可选值: feat, fix, docs, style, refactor, test, chore
2. scope should be the affected module name
3. subject uses imperative mood, lowercase first letter, no period
4. body explains what changed and why
5. footer for issue references or breaking change notes

Only output the commit message, nothing else.
    `;

    const response = await this.openai.chat.completions.create({
      model: 'gpt-4',
      messages: [
        {
          role: 'system',
          content: 'You are a Git expert, skilled at writing standardized commit messages.'
        },
        {
          role: 'user',
          content: prompt
        }
      ],
      temperature: 0.3
    });

    return response.choices[0].message.content.trim();
  }
}

// Usage example
const generator = new CommitMessageGenerator(process.env.OPENAI_API_KEY);
const diff = await execAsync('git diff --staged');
const message = await generator.generate(diff);
await execAsync(`git commit -m "${message}"`);
```

## 3. Automated Branch Management

### 3.1 Branch Naming Conventions

```
AI-suggested branch naming:

Feature branch:
- Input: Implement user login
- Output: feat/user-login

Fix branch:
- Input: Fix login page style issue
- Output: fix/login-page-style

Refactor branch:
- Input: Refactor user authentication module
- Output: refactor/auth-module
```

### 3.2 Intelligent Branch Management

```typescript
class BranchManager {
  async suggestBranchName(task: string): Promise<string> {
    const prompt = `
Based on the following task description, suggest a branch name
following Git Flow conventions.

Task: ${task}

Requirements:
1. Use kebab-case (lowercase with hyphens)
2. Prefix options: feat/, fix/, docs/, refactor/, test/
3. Name should be concise and clear, no more than 50 characters

Only output the branch name, nothing else.
    `;

    // Call AI for suggestion
    const branchName = await this.aiComplete(prompt);
    return branchName.trim();
  }

  async suggestMergeStrategy(
    sourceBranch: string,
    targetBranch: string
  ): Promise<MergeStrategy> {
    const prompt = `
Analyze the following branch merge scenario and suggest the best
merge strategy.

Source branch: ${sourceBranch}
Target branch: ${targetBranch}

Consider:
1. Branch type (feature/fix/release)
2. Whether to preserve commit history
3. Whether to squash commits

Output format:
strategy: merge | squash | rebase
reason: reason for selection
    `;

    const response = await this.aiComplete(prompt);
    return this.parseStrategy(response);
  }
}
```

## 4. AI Code Review Integration

### 4.1 Review Process

```
Pre-commit code review process:

┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  Code    │───→│   AI     │───→│   Fix    │───→│  Manual  │
│ Changes  │    │Pre-review│    │ Issues   │    │  Review  │
│          │    │          │    │          │    │          │
└──────────┘    └────┬─────┘    └──────────┘    └────┬─────┘
                     │                               │
                     ↓                               ↓
               ┌──────────┐                   ┌──────────┐
               │  Issues  │                   │  Pass/   │
               │  Found   │                   │  Reject  │
               └──────────┘                   └──────────┘
```

### 4.2 Git Hook Integration

```bash
#!/bin/sh
# .git/hooks/pre-commit

# Run AI code review
echo "Running AI code review..."

# Get staged files
STAGED_FILES=$(git diff --cached --name-only --diff-filter=ACM | grep -E '\.(js|ts|jsx|tsx)$')

if [ -n "$STAGED_FILES" ]; then
  # Call AI review tool
  npx ai-code-review --files "$STAGED_FILES" --format json > .ai-review-result.json

  # Check for critical issues
  CRITICAL_ISSUES=$(cat .ai-review-result.json | jq '.criticalIssues')

  if [ "$CRITICAL_ISSUES" -gt 0 ]; then
    echo "Found ${CRITICAL_ISSUES} critical issues, please fix before resubmitting"
    cat .ai-review-result.json | jq '.issues[] | select(.severity == "critical")'
    exit 1
  fi

  # Show warnings
  WARNINGS=$(cat .ai-review-result.json | jq '.warnings')
  if [ "$WARNINGS" -gt 0 ]; then
    echo "Found ${WARNINGS} warnings"
  fi
fi

echo "AI review passed"
exit 0
```

### 4.3 PR Review Automation

```yaml
# .github/workflows/ai-code-review.yml
name: AI Code Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  ai-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Get PR diff
        id: diff
        run: |
          git diff origin/${{ github.base_ref }}...HEAD > pr.diff

      - name: AI Code Review
        id: review
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: |
          npx ai-code-review \
            --diff pr.diff \
            --output review.md

      - name: Post Review Comment
        uses: actions/github-script@v6
        with:
          script: |
            const fs = require('fs');
            const review = fs.readFileSync('review.md', 'utf8');

            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## AI Code Review\n\n${review}`
            });
```

### Core Concepts

1. **AI can greatly improve Git operation efficiency**, from commit message generation to code review

2. **Intelligent commit messages** follow Conventional Commits specifications

- Use AI review as pre-review, human review as final review
- Establish team standards, unify AI tool usage
