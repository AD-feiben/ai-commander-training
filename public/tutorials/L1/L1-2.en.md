# What AI Can vs Cannot Do

## Understanding AI Capability Boundaries

Understanding AI's capability boundaries is a prerequisite for becoming an excellent commander. Over-reliance on AI or excessive skepticism of AI will both affect efficiency.

## Areas Where AI Excels

### 1. Code Generation

AI can quickly generate code based on descriptions:
- Writing functions and classes
- Implementing common algorithms
- Generating template code
- Translating between programming languages

### 2. Knowledge Q&A
- Explaining concepts and principles
- Providing technical advice
- Sharing best practices

### 3. Code Review
- Identifying potential bugs
- Discovering security vulnerabilities
- Providing optimization suggestions

### 4. Documentation Generation
- Generating README files
- Writing API documentation
- Creating comments

## Areas Where AI Struggles

### 1. Understanding Incomplete Requirements

If you don't know what you want, AI can't implement it for you either.

### 2. Cross-file Complex Logic

AI excels at logic within single files, but cross-file dependency management still requires human oversight.

### 3. Business Decisions
- Product direction choices
- Technology selection decisions
- Architecture trade-offs

### 4. Innovative Solutions

AI generates content based on existing data; true innovation still requires human thinking.

## Best Practice: Human-AI Collaboration

```
Human Responsibilities:
├── Define requirements and specifications
├── Make key decisions
├── Review and validate
└── Handle edge cases

AI Responsibilities:
├── Generate code
├── Draft documentation
├── Review code
└── Answer routine questions
```

## Summary

| Let AI Do | Do It Yourself |
|-----------|----------------|
| Repetitive code | Requirements definition |
| Template implementation | Key decisions |
| Documentation drafting | Architecture design |
| Bug fixes | Security audits |
| Code explanation | Innovative thinking |
