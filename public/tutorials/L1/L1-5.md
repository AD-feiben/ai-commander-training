# 写出你的第一个 AI 指令

![第一个 AI 指令](/images/tutorials/first-ai-prompt.svg)

## 开始实践

是时候写出你的第一个 AI 指令了！这是一个简单的练习，帮你理解如何与 AI 高效沟通。

## AI 指令的三要素

一个好的 AI 指令应该包含：

| 要素 | 说明 | 示例 |
|------|------|------|
| **任务** | 要做什么 | 验证邮箱地址 |
| **约束** | 有什么要求 | TypeScript、正则表达式 |
| **格式** | 具体格式 | 返回 boolean、添加 JSDoc |

## 示例：好的指令 vs 不好的指令

### ❌ 不好的指令

```markdown
写个邮箱验证函数
```

**问题：**
- 不知道用什么编程语言
- 不知道返回什么格式
- 不知道有什么具体要求
- AI 会猜测，结果可能不符合预期

### ✅ 好的指令

```markdown
帮我写一个 TypeScript 函数，用于验证邮箱地址。

要求：
1. 使用正则表达式验证
2. 返回 boolean
3. 函数名为 isValidEmail
4. 添加 JSDoc 注释
5. 验证规则：标准邮箱格式（包含 @ 和域名）

示例：
- isValidEmail("test@example.com") → true
- isValidEmail("invalid-email") → false
```

**优点：**
- 语言明确（TypeScript）
- 实现方式明确（正则表达式）
- 返回值明确（boolean）
- 命名规范（isValidEmail）
- 有示例说明

## 进阶：完整的函数规范

```markdown
# 邮箱验证函数

## 功能
验证邮箱地址格式是否正确

## 函数签名
```typescript
function isValidEmail(email: string): boolean
```

## 参数
- email: string - 要验证的邮箱地址

## 返回值
- boolean - 格式正确返回 true，否则返回 false

## 验证规则
1. 必须包含 @ 符号
2. @ 前后不能为空
3. 域名部分必须包含至少一个点
4. 点前后不能为空
5. 总长度不超过 254 字符

## 示例
```typescript
isValidEmail("user@example.com")      // true
isValidEmail("user.name@domain.co")   // true
isValidEmail("invalid-email")         // false
isValidEmail("@example.com")          // false
isValidEmail("user@")                 // false
```

## 注意事项
- 不验证邮箱是否真实存在
- 不验证域名是否可解析
- 仅验证格式是否符合标准
```

## 练习任务

### 任务 1：基础练习

写一个简单的 AI 指令，让 AI 生成一个函数：

```markdown
写一个 [语言] 函数，[功能描述]

要求：
1. [要求 1]
2. [要求 2]
3. [要求 3]
```

**示例：**
```markdown
写一个 JavaScript 函数，计算两个日期之间的天数差。

要求：
1. 函数名为 daysBetween
2. 接收两个 Date 参数
3. 返回数字（天数）
4. 处理无效日期输入，返回 null
```

### 任务 2：组件开发

写一个 AI 指令，让 AI 生成一个 React 组件：

```markdown
创建一个 React 组件，[组件功能]

要求：
- 使用 TypeScript
- 使用函数组件
- Props 类型定义完整
- 添加基础样式
- 处理边界情况
```

**示例：**
```markdown
创建一个 React 按钮组件 Button

要求：
- 使用 TypeScript
- 支持 variant: 'primary' | 'secondary' | 'danger'
- 支持 size: 'sm' | 'md' | 'lg'
- 支持 disabled 状态
- 使用 Tailwind CSS 样式
- 添加点击事件处理
```

### 任务 3：完整功能

写一个 AI 指令，让 AI 实现一个完整功能：

```markdown
实现 [功能名称]

## 功能描述
[详细描述]

## 技术要求
- [技术 1]
- [技术 2]

## 验收标准
- [标准 1]
- [标准 2]
```

## 指令优化技巧

### 技巧 1：提供示例

```markdown
❌ 没有示例
"解析日期字符串"

✅ 有示例
"解析日期字符串，格式为 YYYY-MM-DD
示例：
- '2025-01-15' → Date 对象
- 'invalid' → null"
```

### 技巧 2：明确边界条件

```markdown
❌ 不考虑边界
"计算数组平均值"

✅ 考虑边界
"计算数组平均值
- 空数组返回 0
- 包含非数字元素时跳过"
```

### 技巧 3：指定输出格式

```markdown
❌ 不指定格式
"分析这段代码"

✅ 指定格式
"分析这段代码，按以下格式输出：
1. 功能说明
2. 潜在问题
3. 优化建议"
```

## 常见错误

### 错误 1：太模糊

```markdown
❌ "帮我写代码"

✅ "帮我写一个 Python 函数，将 CSV 文件转换为 JSON"
```

### 错误 2：一次要求太多

```markdown
❌ "帮我做一个完整的电商网站"

✅ "帮我实现用户注册功能，包含邮箱验证"
```

### 错误 3：没有约束

```markdown
❌ "写个排序算法"

✅ "写个快速排序算法，要求：
- 时间复杂度 O(n log n)
- 原地排序，不使用额外数组
- 处理重复元素"
```
