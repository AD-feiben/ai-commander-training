# SDD: Spec-driven Development

## What is SDD?

**SDD (Spec-driven Development)** is a development methodology for the AI era. Its core idea is:

> **Specifications are first-class citizens; code is derived from specifications**

In traditional development, we usually write code first and then "casually" add documentation. Specifications are often seen as a burden.

But in the AI era, this order must be reversed.

## Why is SDD Important?

In 2025, with the rise of AI Agents, SDD has become even more important:

1. **AI needs clear specifications** - Specifications are the only window for AI to understand tasks
2. **Verifiability** - Specifications are the standard for judging whether code is "complete"
3. **Team collaboration** - Specifications are the bridge for team communication

## SDD vs Traditional Development

### Traditional Development Flow

```
Requirements → Code → Test → Documentation
   ↓      ↓      ↓
  Verbal Implementation Manual
  Description Understanding Verification
```

### SDD Development Flow

```
Requirements → Spec → Code → Test
   ↓      ↓      ↓
  Spec   AI   Automatic
  Driven Execution Verification
```

## SDD Core Principles

### 1. Spec First

Before letting AI write code, write the specification first.

### 2. Specs Should Be Specific

Avoid vague descriptions; use precise terminology.

### 3. Specs Should Be Complete

Include inputs, outputs, boundaries, and error handling.

### 4. Specs Should Be Verifiable

Every item in the specification should be verifiable.

## Spec Example

```markdown
# User Login Feature Specification

## Input
- username: string, 4-20 characters
- password: string, at least 8 characters

## Output
- success: boolean
- token: string (JWT)
- user: { id, username, email }

## Constraints
- Passwords must be encrypted for storage
- Error messages should not expose user existence
- Lock for 15 minutes after 5 failed login attempts

## Error Handling
- 400: Parameter error
- 401: Username or password error
- 429: Too many login attempts
```

## The Perfect Combination of SDD + AI

SDD methodology and AI are naturally compatible:

1. **Specs are AI's input** - Clear specs let AI accurately understand tasks
2. **Specs are the verification standard** - Let AI self-check based on specs
3. **Specs are the communication bridge** - Team members reach consensus through specs
