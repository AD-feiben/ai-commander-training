# SDD 规范驱动开发

## 什么是 SDD？

**SDD (Spec-driven Development)** 是一种 AI 时代的开发方法论，其核心思想是：

> **规范是第一公民，代码是从规范派生的**

在传统开发中，我们通常是先写代码，然后补文档。但在 AI 时代，清晰的规范是让 AI 准确执行的前提。

## 为什么 SDD 重要？

2025 年，随着 AI Agent 的兴起，SDD 变得更加重要：

1. **AI 需要清晰的规范** - 规范是 AI 理解任务的唯一窗口
2. **可验证性** - 规范是判断代码是否「完成」的标准
3. **团队协作** - 规范是团队沟通的桥梁

## SDD vs 传统开发

### 传统开发流程

```
需求 → 代码 → 测试 → 文档
   ↓      ↓      ↓
  口头   实现   手动
  描述   理解   验证
```

### SDD 开发流程

```
需求 → 规范 → 代码 → 测试
   ↓      ↓      ↓
  规范   AI   自动验证
  驱动   执行
```

## SDD 核心原则

### 1. 规范先行

在让 AI 写代码之前，先写规范。

### 2. 规范要具体

避免模糊的描述，使用精确的术语。

### 3. 规范要完整

包括输入、输出、边界、错误处理。

### 4. 规范可验证

规范中的每一条都应该可以验证。

## 规范示例

```markdown
# 用户登录功能规范

## 输入
- username: string, 4-20个字符
- password: string, 至少8个字符

## 输出
- success: boolean
- token: string (JWT)
- user: { id, username, email }

## 约束
- 密码必须加密存储
- 错误信息不暴露用户存在性
- 登录失败5次后锁定15分钟

## 错误处理
- 400: 参数错误
- 401: 用户名或密码错误
- 429: 登录尝试过多
```

## SDD + AI 的完美结合

SDD 方法论与 AI 是天然契合的：

1. **规范是 AI 的输入** - 清晰的规范让 AI 准确理解任务
2. **规范是验证的标准** - 可以让 AI 根据规范自查
3. **规范是沟通的桥梁** - 团队成员通过规范达成共识
