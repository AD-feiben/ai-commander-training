# SDD 实战：从需求到代码

![SDD 实战](/images/tutorials/spec-driven-development.svg)

## 本课目标

通过本课，你将学会如何将一个真实需求通过 SDD 方法论转化为可执行的代码。

## 实战案例：待办事项应用

### 需求描述

"做一个待办事项应用，用户可以添加、完成、删除任务。"

### 第一步：需求澄清

在与产品经理沟通后，我们得到更详细的需求：

```markdown
## 待办事项应用需求

### 核心功能
1. 添加任务（文本输入）
2. 标记完成/未完成
3. 删除任务
4. 查看所有任务

### 扩展功能
1. 任务分类（工作/生活/学习）
2. 任务优先级
3. 截止日期
4. 数据持久化

### 技术要求
- 前端：React + TypeScript
- 状态管理：Zustand
- 样式：Tailwind CSS
- 存储：LocalStorage

### 验收标准
- 界面美观，响应式设计
- 操作流畅，有动画反馈
- 数据刷新不丢失
```

### 第二步：编写技术规范

**待办事项应用技术规范**

---
版本: 1.0.0
日期: 2025-01-15
---

#### 1. 项目结构

todo-app/
├── src/
│   ├── components/       # UI 组件
│   │   ├── TodoInput.tsx
│   │   ├── TodoItem.tsx
│   │   ├── TodoList.tsx
│   │   └── TodoFilter.tsx
│   ├── stores/          # 状态管理
│   │   └── todoStore.ts
│   ├── types/           # TypeScript 类型
│   │   └── todo.ts
│   ├── utils/           # 工具函数
│   │   └── storage.ts
│   └── App.tsx
├── package.json
└── tailwind.config.js

#### 2. 数据模型

```typescript
interface Todo {
  id: string;
  text: string;
  completed: boolean;
  category: 'work' | 'life' | 'study';
  priority: 'low' | 'medium' | 'high';
  createdAt: Date;
  dueDate?: Date;
}

interface TodoState {
  todos: Todo[];
  filter: 'all' | 'active' | 'completed';
  categoryFilter: string | null;
}
```

#### 3. 组件规范

**TodoInput**
- 功能：输入新任务
- Props: `onAdd: (text: string, category: string) => void`
- 特性：回车添加、空值验证

**TodoItem**
- 功能：显示单个任务
- Props: `todo: Todo, onToggle: () => void, onDelete: () => void`
- 特性：完成划线动画、删除确认

**TodoList**
- 功能：任务列表展示
- Props: `todos: Todo[], filter: string`
- 特性：空状态提示、过滤显示

#### 4. 状态管理

使用 Zustand 管理：
- 任务列表
- 过滤条件
- 本地存储同步

#### 5. 样式规范

- 使用 Tailwind CSS
- 主色调：蓝色 (#3b82f6)
- 响应式：移动端优先
- 动画：使用 Tailwind transitions

### 第三步：让 AI 生成代码

将规范提供给 AI：

```
请根据以下规范实现待办事项应用：

[粘贴上面的规范]

请按以下步骤实现：
1. 创建项目结构和配置文件
2. 实现类型定义
3. 实现状态管理（Zustand store）
4. 实现各个组件
5. 组装 App.tsx
6. 添加样式

每个步骤完成后请告诉我，我会确认后再继续下一步。
```

### 第四步：审查和迭代

AI 生成代码后，进行审查：

```
我检查了代码，有以下问题需要修复：

1. TodoItem 组件缺少完成动画
2. 删除任务前没有确认提示
3. 过滤器在移动端显示不正常

请修复这些问题。
```

### 第五步：验证验收

对照验收标准检查：

- [x] 界面美观，响应式设计
- [x] 操作流畅，有动画反馈
- [x] 数据刷新不丢失
- [x] 可以添加、完成、删除任务
- [x] 支持任务分类和过滤

## SDD 实战要点

### 1. 规范要足够详细

不要假设 AI 知道你的意图，把一切都写清楚。

### 2. 分步骤实现

不要一次性让 AI 实现所有功能，分步骤进行，每步验证。

### 3. 及时反馈

发现问题立即指出，不要让错误累积。

### 4. 保留规范

规范是项目的核心文档，要版本化管理。

## 常见错误

### 错误 1：规范太粗

```
❌ "做个待办应用"

结果：AI 可能做个最简单的版本，没有分类、没有持久化
```

### 错误 2：一次要求太多

```
❌ "实现所有功能"

结果：代码质量差，Bug 多
```

### 错误 3：不验证就继续

```
❌ 直接让 AI 继续下一步，不检查当前步骤

结果：错误累积，最后需要重写
```
