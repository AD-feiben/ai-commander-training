# Write Your First AI Command



## Start Practicing

It's time to write your first AI command! This is a simple exercise to help you understand how to communicate efficiently with AI.

## Three Elements of an AI Command

A good AI command should include:

| Element | Description | Example |
|------|------|------|
| **Task** | What to do | Validate email address |
| **Constraints** | What requirements | TypeScript, regular expression |
| **Format** | Specific format | Return boolean, add JSDoc |

## Examples: Good vs Bad Commands

### ❌ Bad Command

```markdown
Write an email validation function
```

**Problems:**
- Don't know what programming language to use
- Don't know what format to return
- Don't know what specific requirements are
- AI will guess, result may not meet expectations

### ✅ Good Command

```markdown
Help me write a TypeScript function to validate email addresses.

Requirements:
1. Use regular expression for validation
2. Return boolean
3. Function name is isValidEmail
4. Add JSDoc comments
5. Validation rules: Standard email format (containing @ and domain)

Examples:
- isValidEmail("test@example.com") → true
- isValidEmail("invalid-email") → false
```

**Advantages:**
- Language specified (TypeScript)
- Implementation method specified (regular expression)
- Return value specified (boolean)
- Naming convention specified (isValidEmail)
- Has examples to illustrate

## Advanced: Complete Function Specification

```markdown
# Email Validation Function

## Functionality
Validate whether an email address format is correct

## Function Signature
```typescript
function isValidEmail(email: string): boolean
```

## Parameters
- email: string - The email address to validate

## Return Value
- boolean - Returns true if format is correct, otherwise false

## Validation Rules
1. Must contain @ symbol
2. Cannot be empty before or after @
3. Domain must contain at least one dot
4. Cannot be empty before or after dot
5. Total length not exceeding 254 characters

## Examples
```typescript
isValidEmail("user@example.com")      // true
isValidEmail("user.name@domain.co")   // true
isValidEmail("invalid-email")         // false
isValidEmail("@example.com")          // false
isValidEmail("user@")                 // false
```

## Notes
- Does not validate whether the email actually exists
- Does not validate whether the domain is resolvable
- Only validates whether the format meets the standard

## Practice Tasks

### Task 1: Basic Exercise

Write a simple AI command to have AI generate a function:

**Command Template:**

```markdown
Write a [language] function, [function description]

Requirements:
1. [Requirement 1]
2. [Requirement 2]
3. [Requirement 3]
```

**Example:**

> Write a JavaScript function to calculate the number of days between two dates.
>
> Requirements:
> 1. Function name is daysBetween
> 2. Accept two Date parameters
> 3. Return number (days)
> 4. Handle invalid date input, return null

### Task 2: Component Development

Write an AI command to have AI generate a React component:

**Command Template:**

```markdown
Create a React component, [component functionality]

Requirements:
- Use TypeScript
- Use functional components
- Props type definitions complete
- Add basic styles
- Handle edge cases
```

**Example:**

> Create a React button component Button
>
> Requirements:
> - Use TypeScript
> - Support variant: 'primary' | 'secondary' | 'danger'
> - Support size: 'sm' | 'md' | 'lg'
> - Support disabled state
> - Use Tailwind CSS styles
> - Add click event handling

### Task 3: Complete Feature

Write an AI command to have AI implement a complete feature:

**Command Template:**

```markdown
Implement [feature name]

## Feature Description
[Detailed description]

## Technical Requirements
- [Technical requirement 1]
- [Technical requirement 2]

## Acceptance Criteria
- [Criterion 1]
- [Criterion 2]
```

## Command Optimization Tips

### Tip 1: Provide Examples

```markdown
❌ Without examples
"Parse date string"

✅ With examples
"Parse date string in YYYY-MM-DD format
Examples:
- '2025-01-15' → Date object
- 'invalid' → null"
```

### Tip 2: Specify Edge Conditions

```markdown
❌ Not considering edges
"Calculate average of array"

✅ Considering edges
"Calculate average of array
- Empty array returns 0
- Skip non-numeric elements"
```

### Tip 3: Specify Output Format

```markdown
❌ Not specifying format
"Analyze this code"

✅ Specifying format
"Analyze this code with the following output format:
1. Function description
2. Potential issues
3. Optimization suggestions"
```

## Common Mistakes

### Mistake 1: Too Vague

> ❌ "Help me write code"
>
> ✅ "Help me write a Python function to convert CSV file to JSON"

### Mistake 2: Asking Too Much at Once

> ❌ "Help me build a complete e-commerce website"
>
> ✅ "Help me implement user registration feature including email verification"

### Mistake 3: No Constraints

> ❌ "Write a sorting algorithm"
>
> ✅ "Write a quicksort algorithm with requirements:
> - Time complexity O(n log n)
> - In-place sorting without using extra arrays
> - Handle duplicate elements"
