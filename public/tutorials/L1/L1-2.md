# AI 能做什么 vs 不能做什么

![AI 能力边界](/images/tutorials/ai-capabilities.svg)

## 了解 AI 的能力边界

理解 AI 的能力边界是成为优秀指挥官的前提。过度依赖 AI 或过度怀疑 AI 都会影响效率。

## AI 擅长的领域

### 1. 代码生成

AI 可以根据描述快速生成代码：
- 编写函数和类
- 实现常见算法
- 生成模板代码
- 翻译不同编程语言

### 2. 知识问答
- 解释概念和原理
- 提供技术建议
- 分享最佳实践

### 3. 代码审查
- 识别潜在 bug
- 发现安全漏洞
- 提出优化建议

### 4. 文档生成
- 生成 README
- 编写 API 文档
- 创建注释

## AI 不擅长的领域

### 1. 理解不完整的需求

如果你自己都不清楚要什么，AI 也无法帮你实现。

### 2. 跨文件的复杂逻辑

AI 擅长处理单文件内的逻辑，但跨文件的依赖管理仍需要人类把关。

### 3. 业务决策
- 产品方向选择
- 技术选型决策
- 架构权衡

### 4. 创新性解决方案

AI 基于已有数据生成内容，真正的创新仍需要人类思考。

## 最佳实践：人机协作

```
人类负责：
├── 定义需求和规范
├── 做出关键决策
├── 审查和验证
└── 处理边界情况

AI 负责：
├── 生成代码
├── 文档起草
├── 代码审查
└── 常规问题解答
```

## 总结

| 应该让 AI 做   | 应该自己做   |
|---------------|-------------|
| 重复性代码     | 需求定义     |
| 模板化实现     | 关键决策     |
| 文档起草       | 架构设计     |
| Bug 修复      | 安全审计     |
| 代码解释       | 创新思考     |
