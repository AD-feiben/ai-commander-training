# AI 能做什么 vs 不能做什么

![AI 能力边界](/images/tutorials/ai-capabilities.svg)

## 了解 AI 的能力边界

理解 AI 的能力边界是成为优秀指挥官的前提。**过度依赖 AI 或过度怀疑 AI 都会影响效率**。本章将深入分析 AI 在编程领域的能力边界，帮助你建立正确的人机协作认知。

## AI 擅长的领域

### 1. 代码生成与实现

AI 在代码生成方面表现出色，特别是：

**✅ 擅长的场景：**
- **函数和类实现**：根据文档字符串生成具体实现
- **常见算法**：排序、搜索、数据结构操作
- **模板代码**：CRUD 操作、API 路由、组件模板
- **代码翻译**：不同编程语言间的转换
- **正则表达式**：根据描述生成匹配模式
- **SQL 查询**：根据需求生成数据库查询

**示例：**
```python
# 输入："写一个函数，计算斐波那契数列的第 n 项，要求时间复杂度 O(log n)"
# AI 可以生成矩阵快速幂或通项公式实现

def fibonacci(n):
    """使用矩阵快速幂计算斐波那契数列"""
    if n <= 0:
        return 0
    elif n == 1:
        return 1
    
    def multiply(F, M):
        x = F[0][0] * M[0][0] + F[0][1] * M[1][0]
        y = F[0][0] * M[0][1] + F[0][1] * M[1][1]
        z = F[1][0] * M[0][0] + F[1][1] * M[1][0]
        w = F[1][0] * M[0][1] + F[1][1] * M[1][1]
        F[0][0], F[0][1], F[1][0], F[1][1] = x, y, z, w
    
    def power(F, n):
        M = [[1, 1], [1, 0]]
        for _ in range(2, n + 1):
            multiply(F, M)
    
    F = [[1, 1], [1, 0]]
    power(F, n - 1)
    return F[0][0]
```

### 2. 知识问答与解释

AI 拥有广泛的编程知识：

- **概念解释**：设计模式、算法原理、框架机制
- **技术对比**：React vs Vue、SQL vs NoSQL
- **最佳实践**：代码风格、安全规范、性能优化
- **错误诊断**：根据错误信息提供解决方案

### 3. 代码审查与优化

AI 可以辅助进行：

- **Bug 检测**：空指针、数组越界、资源泄漏
- **安全审计**：SQL 注入、XSS、敏感信息泄露
- **性能分析**：时间/空间复杂度、N+1 查询
- **可读性改进**：命名建议、重构建议

### 4. 文档与测试

- **API 文档**：根据代码生成接口文档
- **README 编写**：项目介绍、安装说明
- **单元测试**：根据函数生成测试用例
- **注释生成**：解释复杂逻辑

## AI 的局限性

### 1. 幻觉问题（Hallucination）

> **幻觉**是生成式 AI 最大的技术瓶颈之一。当模型无法找到确切答案时，会"编造"出看似合理但实际上完全错误的信息。

**常见表现：**
- 生成不存在的 API 或函数
- 提供错误的版本信息
- 编造不存在的库或工具

**示例：**
```python
# ❌ AI 可能生成的错误代码
import tensorflow as tf

# 假设用户问："TensorFlow 3.0 的新特性"
# AI 可能编造不存在的 API
model = tf.keras.Sequential3D()  # 不存在的类
model.compile(optimizer='adam5')   # 不存在的优化器
```

**应对策略：**
- 始终验证 AI 提供的 API 和库
- 查阅官方文档确认版本信息
- 对关键代码进行实际测试

### 2. 上下文窗口限制

AI 模型有**上下文窗口（Context Window）**限制，即一次能处理的文本量：

| 模型 | 上下文窗口 | 大约代码量 |
|------|-----------|-----------|
| GPT-4 | 128K tokens | ~300 页代码 |
| Claude 3 | 200K tokens | ~500 页代码 |
| Claude 3.5 | 200K tokens | ~500 页代码 |

**影响：**
- 无法一次性理解整个大型项目
- 跨文件的复杂依赖关系容易遗漏
- 长对话中早期的上下文可能被"遗忘"

**应对策略：**
- 使用 RAG（检索增强生成）提供相关上下文
- 模块化设计，减少单文件复杂度
- 主动提供关键上下文信息

### 3. 理解不完整的需求

**如果你自己都不清楚要什么，AI 也无法帮你实现。**

AI 擅长执行明确的指令，但不擅长：
- 填补模糊的需求空白
- 预测未说明的业务规则
- 理解隐含的上下文

**示例对比：**

```
❌ 模糊需求：
"做一个用户系统"
→ AI 可能生成：简单的登录注册
→ 实际需要的：OAuth、权限管理、审计日志

✅ 清晰需求：
"实现一个用户认证系统，要求：
- 支持邮箱/手机号注册
- 集成 Google OAuth
- 角色权限管理（admin/user/guest）
- 操作审计日志
- JWT Token 认证"
```

### 4. 跨文件的复杂逻辑

AI 擅长处理单文件内的逻辑，但：
- 跨模块的依赖管理需要人工把关
- 架构层面的决策需要人类经验
- 全局重构需要整体规划

### 5. 业务决策与创新

AI 无法替代人类在以下方面的判断：

| 领域 | 原因 |
|------|------|
| **产品方向** | 需要市场洞察和用户理解 |
| **技术选型** | 需要权衡团队能力、生态、长期维护 |
| **架构设计** | 需要考虑可扩展性、性能、成本 |
| **创新方案** | AI 基于已有数据，真正的创新需要人类思考 |

## 2025 年 AI 编程的新边界

### 能力持续扩展

随着技术进步，AI 的能力边界在扩展：

- **Agent 模式**：AI 可以自主规划和执行多步骤任务
- **工具调用**：通过 MCP 连接外部系统和工具
- **多模态**：理解图片、图表、甚至视频内容
- **长上下文**：Claude 3 支持 200K tokens，相当于整本书

### 仍然存在的挑战

- **复杂推理**：多步骤逻辑推理仍可能出错
- **常识理解**：缺乏人类的常识和直觉
- **情感智能**：无法理解微妙的情感和社会语境
- **价值判断**：无法做出涉及伦理和价值的决策

## 最佳实践：人机协作模型

### 黄金法则

```
人类负责：                    AI 负责：
├── 定义需求和规范            ├── 生成代码实现
├── 做出关键决策              ├── 起草文档和测试
├── 架构设计和审查            ├── 代码审查建议
├── 处理边界情况              ├── 常规问题解答
└── 创新和突破               └── 重复性工作
```

### 协作流程建议

```
1. 需求分析（人类主导）
   ↓
2. 编写 Spec（人类编写）
   ↓
3. AI 生成代码（AI 执行）
   ↓
4. 代码审查（人类审查）
   ↓
5. 测试验证（人机协作）
   ↓
6. 迭代优化（人机协作）
```

### 决策树：什么时候用 AI？

```
任务类型判断：
│
├─ 是否涉及业务决策？
│  ├─ 是 → 人类主导
│  └─ 否 → 继续判断
│
├─ 是否是重复性/模板化工作？
│  ├─ 是 → 交给 AI
│  └─ 否 → 继续判断
│
├─ 是否需要创新/突破？
│  ├─ 是 → 人类主导，AI 辅助
│  └─ 否 → 继续判断
│
└─ 是否涉及安全/关键逻辑？
   ├─ 是 → AI 辅助，人类审查
   └─ 否 → 可以交给 AI
```

## 总结：人机分工对照表

| 任务类型 | 建议做法 | 原因 |
|---------|---------|------|
| 重复性代码 | ✅ 让 AI 做 | 提高效率，减少错误 |
| 模板化实现 | ✅ 让 AI 做 | AI 擅长模式匹配 |
| 文档起草 | ✅ 让 AI 做 | 快速生成，人类润色 |
| Bug 修复 | ⚠️ AI 辅助 | AI 提供建议，人类验证 |
| 代码解释 | ✅ 让 AI 做 | AI 知识面广 |
| 需求定义 | ❌ 自己做 | 需要业务理解 |
| 关键决策 | ❌ 自己做 | 需要经验和判断 |
| 架构设计 | ❌ 自己做 | 需要全局视野 |
| 安全审计 | ⚠️ AI 辅助 | AI 发现线索，人类决策 |
| 创新思考 | ❌ 自己做 | AI 基于已有数据 |

## 给 AI 指挥官的建议

1. **保持批判性思维**：不要盲目相信 AI 的输出
2. **建立验证机制**：关键代码必须测试和审查
3. **持续学习**：了解 AI 能力的最新进展
4. **积累经验**：记录 AI 成功和失败的案例
5. **人机互补**：发挥各自优势，形成协作闭环
