# 规范的重要性：代码是规范的马后炮

## 重新认识规范

在传统开发中，我们经常是先写代码，然后「顺便」补文档。规范往往被视为负担。

但在 AI 时代，这个顺序必须颠倒过来。

## 为什么规范是第一位的？

### 1. 规范是 AI 理解任务的窗口

AI 没有「读心术」，它只能看到你给出的信息。规范的清晰度直接决定了 AI 的输出质量。

### 2. 规范是验证的标准

有了规范，我们可以：
- 让 AI 根据规范自检
- 明确判断代码是否「完成」
- 减少返工

### 3. 规范是团队沟通的桥梁

```
人类对话（模糊）：
"做一个用户登录"
↓
规范（清晰）：
"登录需要用户名密码，返回 JWT"
↓
代码（实现）：
```

## 代码是规范的马后炮

这句话的意思是：

> 代码只是规范的「实现」，规范才是「本质」

如果规范错了，代码写得再好也是错的。

如果规范清楚了，代码生成就是水到渠成的事。

## 一个反例

```
需求：做个登录页面

代码实现了：
- 用户名输入
- 密码输入
- 登录按钮
- 错误提示
- 记住我
- 验证码
- 第三方登录
- ...

问题是：需求里根本没提这些！
```

## 正确的做法

```
需求：做个登录页面

规范：
1. 用户名密码登录
2. 错误提示
3. 登录成功后跳转首页

代码：按规范实现，不多不少
```

## 总结

| 传统思维 | AI 思维 |
|---------|---------|
| 代码优先 | 规范优先 |
| 代码即设计 | 规范即设计 |
| 代码是产品 | 规范是产品 |
