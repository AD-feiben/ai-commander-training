# How to Write Good Technical Specifications

## Core Elements of Specifications

A good technical specification should contain the following core elements:

## 1. Functional Description

Concisely describe what this feature does.

```markdown
## Function
Users can log in to the system via username and password
```

## 2. Input Definition

Clearly define input parameters:

```markdown
## Input

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| username | string | Yes | Username, 4-20 characters |
| password | string | Yes | Password, at least 8 characters |
```

## 3. Output Definition

Clearly define output results:

```markdown
## Output

| Field | Type | Description |
|-------|------|-------------|
| success | boolean | Whether successful |
| token | string | JWT token |
| user | object | User information |
```

## 4. Boundary Conditions

Don't ignore edge cases:

```markdown
## Boundary Conditions
1. Username is case-insensitive
2. Wrong password doesn't prompt "user doesn't exist"
3. Lock for 15 minutes after 5 failed login attempts
```

## 5. Error Handling

Define all possible errors:

```markdown
## Error Codes
- 400: Parameter format error
- 401: Username or password error
- 429: Too many login attempts
- 500: Server error
```

## Specification Template

```markdown
# [Feature Name]

## Functional Description
[Brief description]

## Input

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| ... | ... | ... | ... |

## Output

| Field | Type | Description |
|-------|------|-------------|
| ... | ... | ... |

## Boundary Conditions
1. ...
2. ...

## Error Handling
- 400: ...
- 401: ...

## Constraints
- [Performance requirements]
- [Security requirements]
- [Compatibility requirements]
```
