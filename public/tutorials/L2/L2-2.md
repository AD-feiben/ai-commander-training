# 如何写好技术规范

## 规范的核心要素

一份好的技术规范应该包含以下核心要素：

## 1. 功能描述

简明扼要地描述这个功能是什么。

```markdown
## 功能
用户可以通过用户名和密码登录系统
```

## 2. 输入定义

明确输入的参数：

```markdown
## 输入

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，4-20字符 |
| password | string | 是 | 密码，至少8字符 |
```

## 3. 输出定义

明确输出的结果：

```markdown
## 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 是否成功 |
| token | string | JWT令牌 |
| user | object | 用户信息 |
```

## 4. 边界条件

不要忽略边界情况：

```markdown
## 边界条件
1. 用户名不区分大小写
2. 密码错误不提示「用户不存在」
3. 登录失败5次后锁定15分钟
```

## 5. 错误处理

定义所有可能的错误：

```markdown
## 错误码
- 400: 参数格式错误
- 401: 用户名或密码错误
- 429: 登录尝试过多
- 500: 服务器错误
```

## 规范模板

```markdown
# [功能名称]

## 功能描述
[简明扼要描述]

## 输入

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ... | ... | ... | ... |

## 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| ... | ... | ... |

## 边界条件
1. ...
2. ...

## 错误处理
- 400: ...
- 401: ...

## 约束
- [性能要求]
- [安全要求]
- [兼容性要求]
```
