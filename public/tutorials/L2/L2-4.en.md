# AI-Assisted Spec Review

## AI Can Also Review Specifications

You can not only generate code, but also let AI help you review and optimize specifications.

## Review Prompt

```markdown
Please review the following technical specification and point out potential issues:

## Specification Content
[Your specification]

## Please Check
1. Whether it's clear and unambiguous
2. Whether boundary conditions are complete
3. Whether error handling is sufficient
4. Whether there are missing scenarios
5. Whether the specification is verifiable

## Output Format
- Problem list (each containing location, problem, suggestion)
- Improved specification version
```

## Example

Suppose we have this specification:

```markdown
# User Registration Feature

## Input
- username
- password
- email
```

Let AI review it:

```markdown
Please review the following specification:

# User Registration Feature

## Input
- username
- password
- email

---

## AI's Response

### Problems Found

1. **Missing Type Definitions**
   - username, password, email have no type definitions

2. **Missing Length Limits**
   - username and password have no length limits

3. **Missing Format Validation**
   - email has no format requirements

4. **Missing Required Indication**
   - Unclear if all are required

### Improvement Suggestions

Suggested change to:

## Input

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| username | string | Yes | 4-20 characters, alphanumeric and underscore |
| password | string | Yes | At least 8 characters |
| email | string | Yes | Valid email format |
```

## Automated Review Process

```
Write spec → AI reviews → Modify → AI reviews again → ... → Finalize
```

Each iteration makes the specification more complete.
