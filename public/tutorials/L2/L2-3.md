# 规范模板与案例

## 常用规范模板

不同场景需要不同类型的规范模板：

## 1. API 规范模板

```markdown
# 用户登录 API

## 接口
POST /api/auth/login

## 请求头
Content-Type: application/json

## 请求体
{
  "username": "string",
  "password": "string"
}

## 响应 200
{
  "success": true,
  "data": {
    "token": "eyJhbG...",
    "user": {
      "id": "u123",
      "username": "john",
      "email": "john@example.com"
    }
  }
}

## 错误响应
- 400: 参数错误
- 401: 用户名或密码错误
- 429: 请求过于频繁
```

## 2. 函数规范模板

```markdown
# 函数：calculateDiscount

## 功能
计算订单折扣金额

## 输入
- orderAmount: number (订单金额，> 0)
- userLevel: 'normal' | 'silver' | 'gold' (用户等级)

## 输出
- discount: number (折扣金额)
- finalAmount: number (最终金额)

## 规则
- 普通用户：无折扣
- 银卡用户：9折
- 金卡用户：8折

## 边界
- 订单金额 <= 0: 返回错误
- userLevel 无效: 视为 normal
```

## 3. 项目规范模板

```markdown
# 项目：用户管理系统

## 目标
构建一个完整的用户管理系统

## 功能列表
1. 用户注册
2. 用户登录
3. 用户信息修改
4. 密码重置
5. 用户列表查询

## 技术栈
- 前端: React + TypeScript
- 后端: Node.js + Express
- 数据库: PostgreSQL

## 模块划分
- auth: 认证模块
- user: 用户模块
- admin: 管理模块

## 数据模型
- User: id, username, email, password_hash, created_at
```
