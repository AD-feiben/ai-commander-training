# Specification Templates & Examples

## Common Specification Templates

Different scenarios require different types of specification templates:

## 1. API Specification Template

```markdown
# User Login API

## Endpoint
POST /api/auth/login

## Headers
Content-Type: application/json

## Request Body
{
  "username": "string",
  "password": "string"
}

## Response 200
{
  "success": true,
  "data": {
    "token": "eyJhbG...",
    "user": {
      "id": "u123",
      "username": "john",
      "email": "john@example.com"
    }
  }
}

## Error Responses
- 400: Parameter error
- 401: Username or password error
- 429: Too many requests
```

## 2. Function Specification Template

```markdown
# Function: calculateDiscount

## Function
Calculate order discount amount

## Input
- orderAmount: number (order amount, > 0)
- userLevel: 'normal' | 'silver' | 'gold' (user level)

## Output
- discount: number (discount amount)
- finalAmount: number (final amount)

## Rules
- Normal users: no discount
- Silver users: 10% off
- Gold users: 20% off

## Boundaries
- orderAmount <= 0: return error
- Invalid userLevel: treat as normal
```

## 3. Project Specification Template

```markdown
# Project: User Management System

## Goal
Build a complete user management system

## Feature List
1. User registration
2. User login
3. User info modification
4. Password reset
5. User list query

## Tech Stack
- Frontend: React + TypeScript
- Backend: Node.js + Express
- Database: PostgreSQL

## Module Division
- auth: Authentication module
- user: User module
- admin: Admin module

## Data Model
- User: id, username, email, password_hash, created_at
```
