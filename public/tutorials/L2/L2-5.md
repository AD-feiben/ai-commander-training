# 实战：写一个完整的功能规范

## 练习目标

本节将带你完整地编写一个功能规范，使用我们学到的所有知识。

## 需求

```
做一个文件上传功能：
- 支持拖拽上传
- 限制文件大小 10MB
- 支持图片和文档
- 显示上传进度
- 上传成功后显示预览
```

## 第一步：分析需求

提取关键点：
1. 拖拽上传
2. 文件大小限制 10MB
3. 文件类型限制（图片、文档）
4. 进度显示
5. 上传成功预览

## 第二步：编写规范

```markdown
# 文件上传功能规范

## 功能描述
用户可以通过拖拽或点击选择文件上传，支持图片和文档类型

## 输入

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | 要上传的文件 |

## 约束
- 文件大小：≤ 10MB
- 文件类型：image/*, .pdf, .doc, .docx
- 最大并发上传：3个

## 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 是否成功 |
| fileUrl | string | 文件访问 URL |
| fileName | string | 文件名 |
| fileSize | number | 文件大小(字节) |
| fileType | string | 文件 MIME 类型 |
| previewUrl | string | 预览 URL（仅图片）|

## 交互流程
1. 用户拖拽文件到上传区域
2. 显示文件验证（类型、大小）
3. 显示上传进度条
4. 上传成功显示预览
5. 上传失败显示错误信息

## 边界条件
- 文件过大：提示"文件大小超过10MB"
- 文件类型不支持：提示"不支持此文件类型"
- 网络错误：提示"上传失败，请重试"
- 上传中切换页面：继续上传

## 错误码
- 400: 文件验证失败
- 413: 文件过大
- 415: 不支持的类型
- 500: 服务器错误
```

## 第三步：让 AI 审查

将规范发给 AI，让它指出问题：

```markdown
请审查以下规范，指出可能遗漏的边界情况：

[粘贴上面的规范]
```

## 总结

编写规范是一个迭代过程：
1. 先写出初稿
2. 让 AI 审查
3. 根据反馈改进
4. 最终定稿
