# Practice: Write a Complete Feature Specification



## Practice Objectives

This section will guide you to completely write a feature specification using all the knowledge we've learned. Through hands-on practice, you will master the complete process of writing specifications.

## Case: File Upload Feature

### Original Requirements

```
Build a file upload feature:
- Support drag-and-drop upload
- Limit file size to 10MB
- Support images and documents
- Show upload progress
- Show preview after successful upload
```

## Step 1: Requirements Analysis

### Extract Feature Points

| Requirement | Feature Point | Priority |
|------|--------|--------|
| Drag-and-drop upload | Drag interaction, file selection | P0 |
| Size limit 10MB | File validation | P0 |
| Images and documents | File type filtering | P0 |
| Show progress | Progress feedback | P0 |
| Show preview | Preview feature | P1 |

### Identify Implicit Requirements

- Need to validate files before uploading
- Need error handling (network, server errors)
- Support cancel upload
- Multiple file upload
- Upload history

## Step 2: Write Technical Specification

```markdown
# File Upload Feature Specification

---
Version: 1.0.0
Date: 2025-01-15
Status: In Review
---

## 1. Feature Overview

### Background
Users need to upload images and documents to the system for content management and sharing.

### Goal
Provide a simple and intuitive file upload experience, supporting drag-and-drop and click-to-select.

## 2. Functional Requirements

### 2.1 Core Features

#### File Selection
- Support click to select files
- Support drag files to upload area
- Support multiple files simultaneously (max 5)

#### File Validation
- File size: ≤ 10MB
- File types:
  - Images: image/jpeg, image/png, image/gif, image/webp
  - Documents: application/pdf, .doc, .docx
- File name length: ≤ 100 characters

#### Upload Process
- Show upload progress bar (percentage)
- Show upload speed
- Support cancel upload
- Max concurrent uploads: 3

#### Upload Results
- Success: Show file preview (images) or icon (documents)
- Failure: Show error message, support retry
- Provide file access link

### 2.2 User Interface

#### Upload Area
- Default state: Show drag hint and select button
- Dragging: Highlight display
- Validating: Show validation progress
- Uploading: Show progress bar
- Complete: Show preview and success message

#### File List
- Show file name, size, status
- Support delete uploaded files
- Support re-upload failed files

## 3. Technical Specification

### 3.1 Component Design

```typescript
// UploadComponent Props
interface UploadProps {
  accept?: string[];           // Allowed file types
  maxSize?: number;            // Max file size (MB)
  maxCount?: number;           // Max file count
  multiple?: boolean;          // Whether to support multiple files
  onUpload?: (files: File[]) => Promise<UploadResult[]>;
  onError?: (error: UploadError) => void;
  onProgress?: (progress: number) => void;
}

// Upload result
interface UploadResult {
  id: string;
  fileName: string;
  fileSize: number;
  fileType: string;
  url: string;
  previewUrl?: string;         // Image preview URL
  status: 'success' | 'error';
  errorMessage?: string;
}
```

### 3.2 API Interface

#### Upload File
```
POST /api/upload
Content-Type: multipart/form-data

Request:
- file: File (form-data)

Response (200):
{
  "success": true,
  "data": {
    "id": "uuid",
    "fileName": "example.jpg",
    "fileSize": 1024000,
    "fileType": "image/jpeg",
    "url": "https://cdn.example.com/files/uuid.jpg",
    "previewUrl": "https://cdn.example.com/preview/uuid.jpg"
  }
}

Response (400):
{
  "success": false,
  "error": {
    "code": "INVALID_FILE",
    "message": "File validation failed",
    "details": {
      "reason": "FILE_TOO_LARGE",
      "maxSize": "10MB",
      "actualSize": "15MB"
    }
  }
}
```

### 3.3 State Management

```typescript
interface UploadState {
  files: UploadFile[];
  uploading: boolean;
  progress: number;
  error: UploadError | null;
}

interface UploadFile {
  id: string;
  file: File;
  status: 'pending' | 'uploading' | 'success' | 'error';
  progress: number;
  result?: UploadResult;
  error?: UploadError;
}
```

## 4. Edge Cases

### 4.1 File Validation
- Empty file: Prompt "File cannot be empty"
- File too large: Prompt "File size exceeds 10MB limit"
- Unsupported file type: Prompt "This file type is not supported"
- File name too long: Auto-truncate or prompt to rename

### 4.2 Upload Process
- Network interruption: Auto retry 3 times, prompt to retry after failure
- Server error: Prompt "Upload failed, please try again later"
- User cancel: Immediately stop upload, clean up resources
- Page close: Prompt "Upload not complete, are you sure you want to leave?"

### 4.3 Concurrency Control
- Exceed max concurrency: Queue and wait
- User uploads more files: Add to queue

## 5. Performance Requirements

- Upload speed: Fully utilize bandwidth
- Memory usage: Chunked upload for large files to avoid memory overflow
- Response time: Upload interface response < 200ms

## 6. Security Requirements

- File type whitelist: Only allow specified types
- File size limit: Prevent oversized file attacks
- File name cleanup: Prevent path traversal attacks
- Virus scanning: Server-side scanning (optional)

## 7. Error Handling

| Error Code | Scenario | User Prompt |
|--------|------|----------|
| 400 | File validation failed | Show specific error reason |
| 413 | File too large | "File size exceeds 10MB limit" |
| 415 | Unsupported type | "This file type is not supported" |
| 429 | Too many requests | "Please try again later" |
| 500 | Server error | "Upload failed, please try again later" |
| Network error | Network interruption | "Network error, please check network connection" |

## 8. Acceptance Criteria

- [ ] Support drag-and-drop and click to select files
- [ ] Correctly validate file types and sizes
- [ ] Show upload progress bar
- [ ] Support cancel upload
- [ ] Images show preview, documents show icon
- [ ] Correctly handle all error situations
- [ ] Beautiful interface, responsive design
- [ ] Code passes code review
- [ ] Unit test coverage > 80%
```

## Step 3: AI Review

Send the specification to AI for review:

```markdown
Please review the following file upload feature specification, checking from these dimensions:

1. Functionality completeness
2. Edge cases
3. Technical feasibility
4. Security
5. User experience

[Paste the specification above]

Please output:
1. Issues found (classified by severity)
2. Improvement suggestions
3. Optimized specification
```

### Possible AI Feedback

```markdown
## Review Results

### High Priority

1. **Missing resumable upload**
   - When uploading large files, network interruption requires re-upload
   - Suggestion: Add chunked upload and resumable upload feature

2. **Missing upload history**
   - Users may need to view previously uploaded files
   - Suggestion: Add upload history record feature

### Medium Priority

3. **Image compression**
   - Large images upload slowly, occupy large storage space
   - Suggestion: Compress images on client before uploading

4. **Duplicate file detection**
   - Users may upload the same file repeatedly
   - Suggestion: Detect duplicates based on file hash

### Well Considered

✅ File validation (type, size)
✅ Error handling
✅ Concurrency control
✅ Security requirements
```

## Step 4: Iterate and Optimize

Based on AI feedback, optimize the specification:

```markdown
# Optimized Specification (Excerpt)

## New: Chunked Upload

### Large File Handling
- Enable chunked upload when file > 5MB
- Chunk size: 2MB
- Support resumable upload
- Auto retry failed chunks

### API Interface

#### Initialize Upload
```
POST /api/upload/init
{
  "fileName": "example.jpg",
  "fileSize": 10485760,
  "fileType": "image/jpeg",
  "chunkSize": 2097152
}

Response:
{
  "uploadId": "uuid",
  "chunks": 5
}
```

#### Upload Chunk
```
POST /api/upload/chunk
Content-Type: multipart/form-data

- uploadId: uuid
- chunkIndex: 0
- chunk: Blob
```

#### Complete Upload
```
POST /api/upload/complete
{
  "uploadId": "uuid"
}
```

## New: Image Compression

### Client-side Compression
- Auto compress when image > 1MB
- Compressed quality: 80%
- Max dimensions: 1920x1080

## New: Duplicate Detection

### Hash-based Detection
- Calculate file MD5
- Check if already exists before uploading
- If exists, directly return existing file link
```

Please let me know after each step is completed, I will review before continuing to the next step.
```

## Practice Summary

### Specification Writing Process

```
1. Requirements Analysis → Extract feature points
2. Write First Draft → Cover all requirements
3. AI Review → Discover issues
4. Iterate & Optimize → Improve specification
5. Specification Finalized → Team confirmation
6. AI Implement → Develop according to specification
```

### Key Points

1. **Start from requirements**: Don't miss implicit requirements
2. **Be detailed**: Every detail must be clear
3. **Complete edge cases**: Consider all exception situations
4. **Verifiable**: Have clear acceptance criteria
5. **Iterate & optimize**: Continuously improve through review
