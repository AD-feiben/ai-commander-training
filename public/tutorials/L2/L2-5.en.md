# Practice: Write a Complete Feature Spec

## Exercise Objective

This section will guide you through writing a complete functional specification using all the knowledge we've learned.

## Requirement

```
Create a file upload feature:
- Support drag-and-drop upload
- Limit file size to 10MB
- Support images and documents
- Show upload progress
- Display preview after successful upload
```

## Step 1: Analyze Requirements

Extract key points:
1. Drag-and-drop upload
2. File size limit 10MB
3. File type restrictions (images, documents)
4. Progress display
5. Preview after successful upload

## Step 2: Write Specification

```markdown
# File Upload Feature Specification

## Functional Description
Users can upload files via drag-and-drop or click selection, supporting image and document types

## Input

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| file | File | Yes | File to upload |

## Constraints
- File size: ≤ 10MB
- File types: image/*, .pdf, .doc, .docx
- Max concurrent uploads: 3

## Output

| Field | Type | Description |
|-------|------|-------------|
| success | boolean | Whether successful |
| fileUrl | string | File access URL |
| fileName | string | File name |
| fileSize | number | File size (bytes) |
| fileType | string | File MIME type |
| previewUrl | string | Preview URL (images only) |

## Interaction Flow
1. User drags file to upload area
2. Display file validation (type, size)
3. Show upload progress bar
4. Display preview after successful upload
5. Show error message if upload fails

## Boundary Conditions
- File too large: prompt "File size exceeds 10MB"
- Unsupported file type: prompt "This file type is not supported"
- Network error: prompt "Upload failed, please retry"
- Uploading while switching pages: continue uploading

## Error Codes
- 400: File validation failed
- 413: File too large
- 415: Unsupported type
- 500: Server error
```

## Step 3: Let AI Review

Send the specification to AI and let it point out issues:

```markdown
Please review the following specification and point out any missing boundary cases:

[Paste the specification above]
```

## Summary

Writing specifications is an iterative process:
1. Write the first draft
2. Let AI review
3. Improve based on feedback
4. Finalize
