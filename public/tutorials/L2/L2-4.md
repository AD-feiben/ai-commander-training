# 让 AI 帮你审查规范

## AI 也能审查规范

你不仅可以生成代码，AI 还可以帮你审查和优化规范。

## 审查提示词

```markdown
请审查以下技术规范，指出可能存在的问题：

## 规范内容
[你的规范]

## 请检查
1. 是否清晰无歧义
2. 边界条件是否完整
3. 错误处理是否充分
4. 是否有遗漏的场景
5. 规范是否可验证

## 输出格式
- 问题列表（每条包含位置、问题、建议）
- 改进后的规范版本
```

## 示例

假设我们有这样一份规范：

```markdown
# 用户注册功能

## 输入
- username
- password
- email
```

让 AI 审查：

```markdown
请审查以下规范：

# 用户注册功能

## 输入
- username
- password
- email

---

## AI 的回复

### 发现的问题

1. **缺少类型定义**
   - username, password, email 都没有定义类型

2. **缺少长度限制**
   - username 和 password 没有长度限制

3. **缺少格式验证**
   - email 没有格式要求

4. **缺少必填说明**
   - 不清楚是否都是必填

### 改进建议

建议改为：

## 输入

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 4-20字符，字母数字下划线 |
| password | string | 是 | 至少8字符 |
| email | string | 是 | 有效邮箱格式 |
```

## 自动化审查流程

```
写规范 → AI 审查 → 修改 → AI 再审查 → ... → 定稿
```

每次迭代都能让规范更完善。
