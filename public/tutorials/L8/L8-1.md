## 一、性能指标

### 1.1 核心性能指标

```
性能指标金字塔：

                    ┌─────────────┐
                    │   用户体验   │  ← FCP, LCP, CLS
                    └──────┬──────┘
                           │
              ┌────────────┴────────────┐
              │        应用性能          │  ← TTI, TBT, Speed Index
              └────────────┬────────────┘
                           │
        ┌──────────────────┴──────────────────┐
        │              资源性能                │  ← 加载时间、缓存命中率
        └──────────────────┬──────────────────┘
                           │
    ┌──────────────────────┴──────────────────────┐
    │                  代码性能                    │  ← 执行时间、内存使用
    └─────────────────────────────────────────────┘
```

### 1.2 指标详解

| 指标 | 全称 | 说明 | 目标值 |
|------|------|------|--------|
| **FCP** | First Contentful Paint | 首次内容绘制 | < 1.8s |
| **LCP** | Largest Contentful Paint | 最大内容绘制 | < 2.5s |
| **FID** | First Input Delay | 首次输入延迟 | < 100ms |
| **CLS** | Cumulative Layout Shift | 累积布局偏移 | < 0.1 |
| **TTFB** | Time to First Byte | 首字节时间 | < 600ms |
| **TTI** | Time to Interactive | 可交互时间 | < 3.8s |

## 二、常见性能瓶颈

### 2.1 前端性能瓶颈

```javascript
// ❌ 大量 DOM 操作
for (let i = 0; i < 1000; i++) {
  document.body.appendChild(createElement());
}

// ✅ 使用 DocumentFragment
const fragment = document.createDocumentFragment();
for (let i = 0; i < 1000; i++) {
  fragment.appendChild(createElement());
}
document.body.appendChild(fragment);

// ❌ 未优化的重渲染
function Component({ items }) {
  return (
    <div>
      {items.map(item => (
        <ExpensiveComponent key={item.id} data={item} />
      ))}
    </div>
  );
}

// ✅ 使用虚拟列表
import { VirtualList } from 'react-window';

function Component({ items }) {
  return (
    <VirtualList
      height={500}
      itemCount={items.length}
      itemSize={50}
      renderItem={({ index, style }) => (
        <div style={style}>
          <ExpensiveComponent data={items[index]} />
        </div>
      )}
    />
  );
}
```

### 2.2 后端性能瓶颈

```javascript
// ❌ N+1 查询问题
const users = await User.findAll();
for (const user of users) {
  const orders = await Order.find({ userId: user.id }); // N 次查询！
  user.orders = orders;
}

// ✅ 使用 JOIN 或预加载
const users = await User.findAll({
  include: [{ model: Order }]
});

// ❌ 阻塞操作
app.get('/api/data', (req, res) => {
  const data = fs.readFileSync('./large-file.json'); // 阻塞！
  res.json(JSON.parse(data));
});

// ✅ 使用流式处理
app.get('/api/data', (req, res) => {
  const stream = fs.createReadStream('./large-file.json');
  stream.pipe(res);
});
```

## 三、性能分析工具

### 3.1 浏览器 DevTools

```
Chrome DevTools Performance 面板：

1. 录制性能分析
   - 打开 DevTools → Performance
   - 点击录制按钮
   - 执行操作
   - 停止录制

2. 分析结果
   - FPS 图表：查看帧率
   - CPU 图表：查看 CPU 使用
   - Network：查看网络请求
   - Main：查看主线程活动

3. 关键指标
   - 长任务（Long Tasks）
   - 强制同步布局（Forced Reflow）
   - 内存泄漏
```

### 3.2 Node.js 性能分析

```javascript
// 使用 clinic.js
const ClinicDoctor = require('@clinic/doctor');
const doctor = new ClinicDoctor();

doctor.collect(['node', 'server.js'], (err, filepath) => {
```

## 四、建立性能基准

### 4.1 基准测试

```javascript
// 使用 benchmark.js
const Benchmark = require('benchmark');

const suite = new Benchmark.Suite();

suite
  .add('Array#indexOf', () => {
    [1, 2, 3].indexOf(2);
  })
  .add('Array#find', () => {
    [1, 2, 3].find(x => x === 2);
  })
  .on('cycle', (event) => {
    console.log(String(event.target));
  })
  .on('complete', function() {
    console.log('Fastest is ' + this.filter('fastest').map('name'));
  })
  .run();

// 使用 performance API
const start = performance.now();
// 执行代码
const end = performance.now();
console.log(`Execution time: ${end - start}ms`);
```
