你是性能优化专家，精通前端和后端性能分析。

分析以下性能数据，识别瓶颈并提供优化建议。

## 分析维度
1. 加载性能（FCP, LCP）
2. 交互性能（TTI, FID）
3. 资源优化（大小、缓存）
4. 代码执行（长任务、主线程）

## 输出格式
### 瓶颈识别
- 问题描述
- 严重程度
- 影响范围

### 优化建议
- 具体措施
- 预期效果
- 实施难度
```

### 2.2 代码级性能分析

```javascript
// 需要分析的代码
function processData(data) {
  const result = [];
  for (let i = 0; i < data.length; i++) {
    for (let j = 0; j < data.length; j++) {
      if (data[i].id === data[j].parentId) {
        result.push({
          ...data[i],
          parent: data[j]
        });
      }
    }
  }
  return result;
}

// AI 分析结果
const analysis = {
  complexity: 'O(n²)',
  issues: [
    {
      type: 'algorithm',
      description: '嵌套循环导致时间复杂度为 O(n²)',
      suggestion: '使用 Map 存储 parent 关系，优化到 O(n)'
    }
  ],
  optimized: `
function processData(data) {
  const parentMap = new Map(data.map(d => [d.id, d]));
  return data.map(item => ({
    ...item,
    parent: parentMap.get(item.parentId)
  }));
}
  `
};
```

## 三、优化建议生成

### 3.1 前端优化建议

```markdown
## 优化建议模板

### 1. 资源优化
- **代码分割**：使用动态导入
- **压缩**：启用 Gzip/Brotli
- **缓存**：配置长期缓存策略

### 2. 渲染优化
- **虚拟列表**：大数据量使用 react-window
- **懒加载**：图片和组件延迟加载
- **防抖节流**：高频事件处理

### 3. 网络优化
- **预加载**：关键资源 prefetch
- **CDN**：静态资源 CDN 加速
- **HTTP/2**：启用多路复用
```

### 3.2 后端优化建议

```markdown
## 数据库优化
- **索引优化**：添加合适的索引
- **查询优化**：避免 SELECT *
- **连接池**：使用连接池管理

## 缓存策略
- **Redis**：热点数据缓存
- **本地缓存**：应用内缓存
- **CDN 缓存**：静态资源缓存

## 异步处理
- **消息队列**：耗时操作异步化
- **批处理**：批量操作减少 IO
```

## 四、实战：AI 性能分析助手

### 4.1 助手设计

```typescript
class PerformanceAnalyzer {
  private openai: OpenAI;
  
  constructor(apiKey: string) {
    this.openai = new OpenAI({ apiKey });
  }
  
  async analyze(data: PerformanceData): Promise<AnalysisResult> {
    const prompt = this.buildPrompt(data);
    
    const response = await this.openai.chat.completions.create({
      model: 'gpt-4',
      messages: [
        {
          role: 'system',
          content: '你是性能优化专家，分析性能数据并提供优化建议。'
        },
        {
          role: 'user',
          content: prompt
        }
      ],
      response_format: { type: 'json_object' }
    });
    
    return JSON.parse(response.choices[0].message.content);
  }
  
  private buildPrompt(data: PerformanceData): string {
    return `
分析以下性能数据：

## 页面指标
- FCP: ${data.fcp}ms
- LCP: ${data.lcp}ms
- TTI: ${data.tti}ms
- CLS: ${data.cls}

## 资源加载
${data.resources.map(r => `- ${r.name}: ${r.size}KB, ${r.time}ms`).join('\n')}

## 长任务
${data.longTasks.map(t => `- ${t.name}: ${t.duration}ms`).join('\n')}

请按 JSON 格式返回：
{
  "bottlenecks": [...],
  "suggestions": [...],
  "priority": "high|medium|low"
}
    `;
  }
}
```

### 4.2 使用示例

```typescript
const analyzer = new PerformanceAnalyzer(process.env.OPENAI_API_KEY);

const result = await analyzer.analyze({
  fcp: 2500,
  lcp: 4200,
  tti: 5800,
  cls: 0.15,
  resources: [
    { name: 'app.js', size: 850, time: 1200 },
    { name: 'vendor.js', size: 1200, time: 1500 }
  ],
  longTasks: [
    { name: 'script-evaluation', duration: 350 }
  ]
});

console.log(result);
```
