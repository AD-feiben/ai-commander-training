## 1. Performance Metrics

### 1.1 Core Performance Metrics

**Performance Metrics Pyramid** (top to bottom):

1. **User Experience Layer** <- FCP, LCP, CLS
2. **Application Performance Layer** <- TTI, TBT, Speed Index
3. **Resource Performance Layer** <- Load time, cache hit rate
4. **Code Performance Layer** <- Execution time, memory usage

### 1.2 Metrics Explained

| Metric | Full Name | Description | Target |
|--------|-----------|-------------|--------|
| **FCP** | First Contentful Paint | First content paint | < 1.8s |
| **LCP** | Largest Contentful Paint | Largest content paint | < 2.5s |
| **FID** | First Input Delay | First input delay | < 100ms |
| **CLS** | Cumulative Layout Shift | Cumulative layout shift | < 0.1 |
| **TTFB** | Time to First Byte | Time to first byte | < 600ms |
| **TTI** | Time to Interactive | Time to interactive | < 3.8s |

## 2. Common Performance Bottlenecks

### 2.1 Frontend Performance Bottlenecks

```javascript
// ❌ Excessive DOM operations
for (let i = 0; i < 1000; i++) {
  document.body.appendChild(createElement());
}

// ✅ Use DocumentFragment
const fragment = document.createDocumentFragment();
for (let i = 0; i < 1000; i++) {
  fragment.appendChild(createElement());
}
document.body.appendChild(fragment);

// ❌ Unoptimized re-renders
function Component({ items }) {
  return (
    <div>
      {items.map(item => (
        <ExpensiveComponent key={item.id} data={item} />
      ))}
    </div>
  );
}

// ✅ Use virtual list
import { VirtualList } from 'react-window';

function Component({ items }) {
  return (
    <VirtualList
      height={500}
      itemCount={items.length}
      itemSize={50}
      renderItem={({ index, style }) => (
        <div style={style}>
          <ExpensiveComponent data={items[index]} />
        </div>
      )}
    />
  );
}
```

### 2.2 Backend Performance Bottlenecks

```javascript
// ❌ N+1 query problem
const users = await User.findAll();
for (const user of users) {
  const orders = await Order.find({ userId: user.id }); // N queries!
  user.orders = orders;
}

// ✅ Use JOIN or eager loading
const users = await User.findAll({
  include: [{ model: Order }]
});

// ❌ Blocking operations
app.get('/api/data', (req, res) => {
  const data = fs.readFileSync('./large-file.json'); // Blocking!
  res.json(JSON.parse(data));
});

// ✅ Use streaming
app.get('/api/data', (req, res) => {
  const stream = fs.createReadStream('./large-file.json');
  stream.pipe(res);
});
```

## 3. Performance Analysis Tools

### 3.1 Browser DevTools

```
Chrome DevTools Performance Panel:

1. Record performance profile
   - Open DevTools → Performance
   - Click record button
   - Perform operations
   - Stop recording

2. Analyze results

```

### 3.2 Node.js Performance Analysis

```javascript
// Using clinic.js
const ClinicDoctor = require('@clinic/doctor');
const doctor = new ClinicDoctor();

doctor.collect(['node', 'server.js'], (err, filepath) => {
  if (err) throw err;
  
  doctor.visualize(filepath, './report.html', (err) => {
    if (err) throw err;
    console.log('Report generated');
  });
});

// Using 0x flamegraph
const zeroEx = require('0x');

zeroEx({
  argv: ['server.js']
}, (err, file) => {
  console.log(`Flamegraph: ${file}`);
});
```

## 4. Establishing Performance Baselines

### 4.1 Benchmark Testing

```javascript
// Using benchmark.js
const Benchmark = require('benchmark');

const suite = new Benchmark.Suite();

suite
  .add('Array#indexOf', () => {
    [1, 2, 3].indexOf(2);
  })
  .add('Array#find', () => {
    [1, 2, 3].find(x => x === 2);
  })
  .on('cycle', (event) => {
    console.log(String(event.target));
  })
  .on('complete', function() {
    console.log('Fastest is ' + this.filter('fastest').map('name'));
  })
  .run();

// Using performance API
const start = performance.now();
// Execute code
const end = performance.now();
console.log(`Execution time: ${end - start}ms`);
```
