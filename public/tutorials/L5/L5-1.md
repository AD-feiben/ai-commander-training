# L5-1: Agent 技能系统概述

> 让 AI 掌握专业技能，成为真正的领域专家

## 本节导读

在前面的章节中，我们学习了如何通过提示词与 AI 进行有效沟通。但如果你想让 AI 在特定领域表现得像真正的专家一样，就需要为它配备**技能（Skills）**——这是 Agent 能够执行专业任务的核心能力。

本节课将带你了解：
- 什么是 Agent 技能系统
- 为什么需要技能系统
- 技能的组成部分和工作原理
- 如何为 AI 配备合适的技能

---

## 一、从通用 AI 到专业 Agent

### 1.1 通用 AI 的局限性

想象你请了一位"全能助手"，他什么都知道一点，但在专业领域总是差点火候：

```
┌─────────────────────────────────────────────────────────────┐
│                    通用 AI 的能力边界                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   基础知识        编程能力        领域专长        工具使用   │
│   ┌─────┐        ┌─────┐        ┌─────┐        ┌─────┐    │
│   │■■■■ │        │■■■■ │        │■■  │        │■    │    │
│   │■■■■ │        │■■■■ │        │■■  │        │■    │    │
│   │■■■■ │        │■■■■ │        │■■  │        │■    │    │
│   │■■■■ │        │■■■■ │        │■■  │        │■    │    │
│   └─────┘        └─────┘        └─────┘        └─────┘    │
│   较强           较强           较弱           较弱        │
│                                                             │
│   问题：缺乏深度领域知识和专业工具调用能力                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**具体表现：**
- 能写代码，但不熟悉特定框架的最佳实践
- 能分析数据，但不懂行业特定的分析方法
- 能回答问题，但无法调用专业工具获取实时信息

### 1.2 Agent 技能系统的价值

**技能系统 = 给 AI 装上"专业装备"**

```
┌─────────────────────────────────────────────────────────────┐
│                  Agent + 技能系统                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                    ┌─────────────┐                          │
│                    │   通用 AI    │                          │
│                    │   基础能力   │                          │
│                    └──────┬──────┘                          │
│                           │                                 │
│              ┌────────────┼────────────┐                    │
│              │            │            │                    │
│         ┌────┴────┐  ┌────┴────┐  ┌────┴────┐              │
│         │ 技能 1  │  │ 技能 2  │  │ 技能 3  │              │
│         │React开发│  │数据库设计│  │API测试  │              │
│         │■■■■■■■■ │  │■■■■■■■■ │  │■■■■■■■■ │              │
│         └─────────┘  └─────────┘  └─────────┘              │
│                                                             │
│   结果：在特定领域达到专家水平，能调用专业工具完成任务          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**核心价值：**

| 维度 | 通用 AI | Agent + 技能 |
|------|---------|--------------|
| **知识深度** | 广泛但浅显 | 在特定领域深入专业 |
| **工具使用** | 无法调用外部工具 | 可调用 API、数据库、文件系统等 |
| **一致性** | 每次回答可能有差异 | 遵循既定规范和流程 |
| **可复用性** | 需要重复说明要求 | 技能一次配置，多次复用 |
| **协作能力** | 单轮对话 | 可与其他 Agent 协作 |

## 二、什么是 Agent 技能

### 2.1 技能的定义

**技能（Skill）** 是 Agent 执行特定任务的能力单元，包含：

```
┌─────────────────────────────────────────────────────────────┐
│                      技能（Skill）                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │   知识库      │  │   工具集      │  │   工作流      │      │
│  │  (Knowledge) │  │   (Tools)    │  │ (Workflow)   │      │
│  ├──────────────┤  ├──────────────┤  ├──────────────┤      │
│  │ • 领域知识    │  │ • API 调用   │  │ • 执行步骤    │      │
│  │ • 最佳实践    │  │ • 文件操作   │  │ • 决策逻辑    │      │
│  │ • 代码模板    │  │ • 数据库查询 │  │ • 错误处理    │      │
│  │ • 规范标准    │  │ • 命令执行   │  │ • 输出格式    │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                    提示词模板                          │  │
│  │              (Prompt Template)                        │  │
│  │    定义如何调用技能、处理输入、格式化输出              │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 技能的分类

**按功能分类：**

| 类型 | 说明 | 示例 |
|------|------|------|
| **开发技能** | 编程相关的专业能力 | React 开发、数据库设计、API 开发 |
| **分析技能** | 数据处理和分析能力 | 日志分析、性能分析、代码审查 |
| **工具技能** | 调用外部工具的能力 | Git 操作、Docker 管理、云服务调用 |
| **领域技能** | 特定行业知识 | 金融计算、医疗数据处理、电商业务 |
| **协作技能** | 多 Agent 协作能力 | 任务分配、结果汇总、冲突解决 |

**按复杂度分类：**

```
简单技能                    中等技能                    复杂技能
┌─────────┐              ┌─────────────┐            ┌─────────────────┐
│ 正则生成 │              │  React 组件  │            │  微服务架构设计  │
│         │              │    开发      │            │                 │
│ 单一步骤 │─────────────→│  多步骤协作  │───────────→│  多 Agent 协作   │
│ 无状态   │              │   有状态     │            │   长期任务管理   │
└─────────┘              └─────────────┘            └─────────────────┘
   1-2 步                    3-10 步                   10+ 步
   即时完成                  分钟级完成                 小时级完成
```

## 三、技能系统的工作原理

### 3.1 技能调用流程

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  用户请求 │────→│ 技能匹配  │────→│ 技能执行  │────→│ 结果返回  │
└──────────┘     └──────────┘     └──────────┘     └──────────┘
                      │                │
                      ↓                ↓
               ┌──────────┐      ┌──────────┐
               │ 意图识别  │      │ 工具调用  │
               │ 参数提取  │      │ 知识检索  │
               └──────────┘      │ 流程执行  │
                                  └──────────┘
```

**详细流程：**

1. **意图识别**：分析用户请求，确定需要调用什么技能
2. **参数提取**：从请求中提取技能所需的参数
3. **技能匹配**：找到最合适的技能（或多个技能组合）
4. **知识检索**：从技能知识库获取相关信息
5. **工具调用**：使用技能的工具集执行操作
6. **流程执行**：按照工作流步骤完成任务
7. **结果格式化**：将结果整理为用户期望的格式

### 3.2 技能配置示例

**以 "React 组件开发技能" 为例：**

```yaml
skill:
  name: react-component-dev
  version: 1.0.0
  description: 专业的 React 组件开发技能
  
  # 知识库
  knowledge:
    - react-best-practices.md
    - component-patterns/
    - hooks-guidelines.md
    - typescript-react.md
  
  # 工具集
  tools:
    - file_reader: 读取项目文件
    - file_writer: 写入生成的代码
    - linter: 运行 ESLint 检查
    - test_runner: 运行单元测试
  
  # 工作流
  workflow:
    steps:
      1. analyze_requirements:
          action: 分析组件需求
          output: requirements.json
      
      2. design_component:
          action: 设计组件结构
          input: requirements.json
          output: component-design.md
      
      3. generate_code:
          action: 生成组件代码
          input: component-design.md
          output: Component.tsx
      
      4. validate_code:
          action: 验证代码质量
          input: Component.tsx
          tools: [linter, test_runner]
      
      5. optimize:
          action: 根据反馈优化
          condition: if validation failed
  
  # 提示词模板
  prompt_template: |
    你是 React 开发专家，使用以下规范生成组件：
    
    ## 需求
    {requirements}
    
    ## 项目规范
    {project_standards}
    
    ## 输出要求
    1. 使用 TypeScript
    2. 遵循函数组件 + Hooks 模式
    3. 包含 PropTypes 定义
    4. 添加 JSDoc 注释
    5. 包含单元测试
```

## 四、主流技能系统架构

### 4.1 MCP (Model Context Protocol)

**MCP** 是 Anthropic 推出的开放协议，让 AI 能够安全地连接外部工具和数据源。

```
┌─────────────────────────────────────────────────────────────┐
│                     MCP 架构                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────┐         ┌─────────────┐                  │
│   │   AI 模型    │←───────→│  MCP 主机    │                  │
│   │  (Claude)   │         │  (Host)     │                  │
│   └─────────────┘         └──────┬──────┘                  │
│                                  │                          │
│                    ┌─────────────┼─────────────┐            │
│                    │             │             │            │
│              ┌─────┴────┐  ┌─────┴────┐  ┌─────┴────┐      │
│              │ MCP 服务 │  │ MCP 服务 │  │ MCP 服务 │      │
│              │ (文件)   │  │ (数据库) │  │ (API)   │      │
│              └─────────┘  └─────────┘  └─────────┘      │
│                                                             │
│   特点：标准化协议、即插即用、安全可靠                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**MCP 的核心优势：**
- **标准化**：统一的协议，不同服务可以互换
- **安全性**：用户控制权限，AI 无法擅自操作
- **扩展性**：轻松添加新的工具和数据源

### 4.2 Function Calling

**Function Calling** 是 OpenAI 推出的机制，让模型可以调用外部函数。

```typescript
// 定义可用函数
const functions = [
  {
    name: "get_weather",
    description: "获取指定城市的天气",
    parameters: {
      type: "object",
      properties: {
        city: { type: "string", description: "城市名称" },
        date: { type: "string", description: "日期 (YYYY-MM-DD)" }
      },
      required: ["city"]
    }
  },
  {
    name: "search_code",
    description: "在代码库中搜索",
    parameters: {
      type: "object",
      properties: {
        query: { type: "string", description: "搜索关键词" },
        file_type: { type: "string", enum: [".ts", ".js", ".tsx"] }
      },
      required: ["query"]
    }
  }
];

// AI 决定调用哪个函数
const response = await openai.chat.completions.create({
  model: "gpt-4",
  messages: userMessages,
  functions: functions,
  function_call: "auto"
});

// 如果 AI 决定调用函数
if (response.choices[0].message.function_call) {
  const functionName = response.choices[0].message.function_call.name;
  const args = JSON.parse(response.choices[0].message.function_call.arguments);
  
  // 执行函数
  const result = await executeFunction(functionName, args);
  
  // 将结果返回给 AI
  const finalResponse = await openai.chat.completions.create({
    model: "gpt-4",
    messages: [
      ...userMessages,
      response.choices[0].message,
      { role: "function", name: functionName, content: JSON.stringify(result) }
    ]
  });
}
```

### 4.3 技能系统对比

| 特性 | MCP | Function Calling | 自定义技能系统 |
|------|-----|------------------|----------------|
| **标准化程度** | 高 | 中 | 低 |
| **易用性** | 中 | 高 | 取决于实现 |
| **灵活性** | 中 | 高 | 极高 |
| **生态支持** |  growing | 成熟 | 无 |
| **适用场景** | 企业集成 | 快速开发 | 深度定制 |

## 五、如何为 AI 配备技能

### 5.1 技能选择策略

**根据任务类型选择：**

```
任务分析
    │
    ├── 需要外部数据？ ──→ 数据查询技能
    │
    ├── 需要代码生成？ ──→ 编程技能（React/Vue/Node等）
    │
    ├── 需要代码审查？ ──→ 代码分析技能
    │
    ├── 需要文档处理？ ──→ 文档生成技能
    │
    └── 需要多步骤协作？ ──→ 工作流编排技能
```

**技能组合示例：**

| 场景 | 技能组合 |
|------|----------|
| 开发新功能 | 需求分析 + React开发 + 测试生成 |
| 代码重构 | 代码分析 + 重构建议 + 测试验证 |
| Bug 修复 | 日志分析 + 问题诊断 + 代码修复 |
| 技术调研 | 信息检索 + 文档生成 + 方案对比 |

### 5.2 技能配置最佳实践

**1. 渐进式添加**
```
阶段 1：基础技能
    └─→ 代码生成、代码审查

阶段 2：专业深化
    └─→ 添加框架特定技能（React、Vue、Node）

阶段 3：工具集成
    └─→ 添加 Git、Docker、CI/CD 技能

阶段 4：领域定制
    └─→ 添加业务领域特定技能
```

**2. 版本管理**
```yaml
skill:
  name: react-dev

    - 2.1.0: 添加 React 18 支持

    - 1.5.0: 添加测试生成能力
```

**3. 效果评估**
```

├── 平均耗时：45 秒
```

## 六、实战：配置一个代码生成技能

### 6.1 技能定义

**目标**：创建一个 TypeScript React 组件生成技能

```yaml
skill:
  name: tsx-component-generator
  display_name: TypeScript React 组件生成器
  version: 1.0.0
  
  description: |
    根据需求生成符合项目规范的 TypeScript React 组件。
    支持函数组件、Hooks、TypeScript 类型定义、样式文件和单元测试。
  
  # 触发条件
  triggers:
    - keywords: ["生成组件", "创建组件", "component"]
    - file_patterns: ["*.tsx", "*.ts"]
  
  # 输入参数
  parameters:
    component_name:
      type: string
      required: true
      description: 组件名称（PascalCase）
    
    props:
      type: array
      required: false
      description: 组件属性列表
      items:
        name: string
        type: string
        required: boolean
        default: any
    
    features:
      type: array
      required: false
      description: 需要的功能特性
      enum: ["state", "effect", "callback", "memo", "ref"]
    
    style_type:
      type: string
      required: false
      default: "css_modules"
      enum: ["css_modules", "styled_components", "tailwind", "inline"]
  
  # 知识库
  knowledge_base:
    files:
      - react-coding-standards.md
      - typescript-best-practices.md
      - component-templates/
    
    rules:
      - 使用函数组件和 Hooks
      - Props 必须定义 TypeScript 接口
      - 复杂逻辑使用自定义 Hook 抽取
      - 组件文件不超过 200 行
  
  # 工作流
  workflow:
    step_1_analyze:
      name: 需求分析
      action: analyze_requirements
      output: analysis_report
    
    step_2_design:
      name: 组件设计
      action: design_component
      input: analysis_report
      output: component_design
    
    step_3_generate:
      name: 代码生成
      action: generate_code
      input: component_design
      output: 
        - Component.tsx
        - Component.module.css
        - Component.test.tsx
        - index.ts
    
    step_4_validate:
      name: 代码验证
      action: validate_code
      tools: [eslint, tsc, jest]
      on_failure: retry_with_fixes
  
  # 输出格式
  output:
    format: structured
    files:
      main: "{component_name}.tsx"
      styles: "{component_name}.module.css"
      test: "{component_name}.test.tsx"
      export: "index.ts"
```

## 七、本节小结

### 核心概念

1. **技能（Skill）** 是 Agent 执行特定任务的能力单元，包含知识库、工具集、工作流和提示词模板

2. **技能系统** 让 AI 从通用助手转变为专业领域的专家，能够调用工具、遵循规范、执行复杂流程

3. **主流架构**：MCP（标准化协议）、Function Calling（函数调用）、自定义技能系统

4. **技能选择** 应根据任务类型、项目需求和团队规范进行合理搭配

### 关键要点

- 技能 = 知识 + 工具 + 流程 + 模板
- 好的技能系统应该可配置、可扩展、可评估
- 渐进式添加技能，从基础到专业
- 持续监控技能使用效果，不断优化
