# 代码审查要点

## AI 生成代码也需要审查

即使是 AI 生成的代码，也需要人工审查。以下是代码审查的核心要点。

## 审查清单

### 1. 安全问题

| 问题 | 检查点 |
|------|--------|
| SQL 注入 | 是否使用参数化查询 |
| XSS | 输出是否转义 |
| 敏感信息 | 是否暴露密钥、密码 |
| 权限 | 是否有越权风险 |

### 2. 性能问题

| 问题 | 检查点 |
|------|--------|
| N+1 查询 | 是否有循环查询 |
| 内存泄漏 | 是否有未释放资源 |
| 大数据 | 是否有分页处理 |
| 缓存 | 是否有缓存策略 |

### 3. 逻辑问题

| 问题 | 检查点 |
|------|--------|
| 边界条件 | 是否处理空值、极限值 |
| 错误处理 | 是否有 try-catch |
| 业务逻辑 | 是否符合需求 |
| 依赖 | 是否处理依赖失败 |

### 4. 代码质量

| 问题 | 检查点 |
|------|--------|
| 命名 | 变量/函数名是否清晰 |
| 注释 | 关键逻辑是否有注释 |
| 复杂度 | 函数是否过长 |
| 重复 | 是否有重复代码 |
