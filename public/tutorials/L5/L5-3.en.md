# L5-3: Knowledge Base Construction & Management

> Building AI's professional knowledge foundation to make skills truly expert in their domains

## Chapter Overview

The knowledge base is the core competitive advantage of Agent skills. An excellent knowledge base can transform AI from a "general assistant" to a "domain expert". This chapter will deeply explain how to build, organize, and manage high-quality knowledge bases.

Through this chapter, you will learn:
- Design principles and organization methods for knowledge bases
- How to collect and organize domain knowledge
- Knowledge retrieval and update strategies
- Practice: Build a frontend development knowledge base

## 2. Design Principles for Knowledge Base

### 2.1 Design Principles Pyramid

```
                    ┌─────────────┐
                    │  Utility    │  ← Solve actual problems
                    └──────┬──────┘
                           │
              ┌────────────┴────────────┐
              │      Maintainability     │  ← Easy to update and maintain
              └────────────┬────────────┘
                           │
        ┌──────────────────┴──────────────────┐
        │             Scalability             │  ← Support new domains
        └──────────────────┬──────────────────┘
                           │
    ┌──────────────────────┴──────────────────────┐
    │                  Structured                 │  ← Clear organization
    └────────────────────────────────────────────┘
```

### 2.2 Six Design Principles

#### Principle 1: MECE (Mutually Exclusive, Collectively Exhaustive)

**Meaning:** Knowledge categorization should be non-overlapping and complete

```
❌ Wrong Example (has overlap):
knowledge-base/
├── frontend/
│   └── React/
├── framework/
│   └── React/  ← Duplicate!
└── performance/
    └── React performance/  ← Duplicate!

✅ Correct Example (MECE):
knowledge-base/
├── by-tech-stack/
│   ├── React/
│   ├── Vue/
│   └── Angular/
├── by-topic/
│   ├── performance/
│   ├── security/
│   └── testing/
└── by-level/
    ├── beginner/
    ├── intermediate/
    └── expert/
```

#### Principle 2: Progressive Detail

**Meaning:** Organize from summary to detail, layered

```
Knowledge Entry Structure:
┌─────────────────────────────────────┐
│ 1. Overview (1-2 sentences)         │
├─────────────────────────────────────┤
│ 2. Why Important (use cases & value) │
├─────────────────────────────────────┤
│ 3. Core Concepts (key terms & princ.)│
├─────────────────────────────────────┤
│ 4. Best Practices (how to do)       │
├─────────────────────────────────────┤
│ 5. Common Pitfalls (cautions)        │
├─────────────────────────────────────┤
│ 6. Example Code (Good & Bad)        │
├─────────────────────────────────────┤
│ 7. Reference Resources              │
└─────────────────────────────────────┘
```

#### Principle 3: Example-Driven

**Meaning:** Each knowledge point should have good/bad examples

```markdown
## Avoiding Memory Leaks

### ❌ Wrong Example
```javascript
function useDataFetching() {
  useEffect(() => {
    const interval = setInterval(() => {
      fetchData();
    }, 1000);
    // Forgot to clean up!
  }, []);
}
```

### ✅ Correct Example
```javascript
function useDataFetching() {
  useEffect(() => {
    const interval = setInterval(() => {
      fetchData();
    }, 1000);
    
    // Cleanup function
    return () => clearInterval(interval);
  }, []);
}
```

### 🔍 Detection Methods
1. React DevTools Profiler
2. Chrome Memory panel
3. Use `why-did-you-render`
```

# useEffect Best Practices

> useEffect is the core Hook for handling side effects in React. Proper usage is crucial for application performance and stability.

## Overview

useEffect allows you to perform side effects in functional components, such as data fetching, subscriptions, and manual DOM mutations.

## Why Important

- **Data Fetching**: Load data from API
- **Subscription Management**: WebSocket, event listeners
- **DOM Manipulation**: Integrate non-React libraries
- **Performance Optimization**: Control unnecessary executions

Improper usage causes:
- Infinite render loops
- Memory leaks
- Race conditions
- Performance issues

## Core Concepts

### Timing

```
Component renders → Browser paints → Execute useEffect
```

### Dependency Array

| Dependency Array | Timing |
|------------------|--------|
| None | Execute after every render |
| `[]` | Execute on mount only |
| `[a, b]` | Mount + when dependencies change |

## Best Practices

### ✅ Set Dependencies Correctly

```javascript
// ✅ Include all dependencies
useEffect(() => {
  const data = fetchData(userId);
  setData(data);
}, [userId]); // Include userId

// ❌ Missing dependency
useEffect(() => {
  const data = fetchData(userId);
  setData(data);
}, []); // Missing userId, causes stale closure
```

### ✅ Use Cleanup Functions

```javascript
// ✅ Subscription with cleanup
useEffect(() => {
  const subscription = subscribeToUser(userId);
  
  return () => {
    subscription.unsubscribe();
  };
}, [userId]);

// ✅ Timer cleanup
useEffect(() => {
  const timer = setInterval(() => {
    console.log('tick');
  }, 1000);
  
  return () => clearInterval(timer);
}, []);
```

### ✅ Separate Unrelated Logic

```javascript
// ❌ One effect doing too much
useEffect(() => {
  fetchUser(userId);
  fetchPosts(userId);
  updateAnalytics();
}, [userId]);

// ✅ Split into separate effects
useEffect(() => {
  fetchUser(userId);
}, [userId]);

useEffect(() => {
  fetchPosts(userId);
}, [userId]);

useEffect(() => {
  updateAnalytics();
}, []);
```

### ✅ Handle Async Operations

```javascript
// ✅ Use IIFE
useEffect(() => {
  let cancelled = false;
  
  (async () => {
    const data = await fetchData(userId);
    
    if (!cancelled) {
      setData(data);
    }
  })();
  
  return () => {
    cancelled = true;
  };
}, [userId]);
```

## Common Pitfalls

| Pitfall | Symptom | Solution |
|---------|---------|----------|
| Missing dependencies | Data not updating, closure issues | Use ESLint exhaustive-deps rule |
| Infinite loop | Page freezes | Check dependency array, avoid setState triggering updates |
| Race condition | Old data overwrites new | Use cancellation flag or AbortController |
| Memory leak | Still executing after unmount | Use cleanup function |

## Practical Examples

### Data Fetching Pattern

```javascript
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    let cancelled = false;
    
    async function fetchUser() {
      try {
        setLoading(true);
        setError(null);
        
        const data = await api.getUser(userId);
        
        if (!cancelled) {
          setUser(data);
        }
      } catch (err) {
        if (!cancelled) {
          setError(err.message);
        }
      } finally {
        if (!cancelled) {
          setLoading(false);
        }
      }
    }
    
    fetchUser();
    
    return () => {
      cancelled = true;
    };
  }, [userId]);

  if (loading) return <Spinner />;
  if (error) return <Error message={error} />;
  return <UserCard user={user} />;
}
```
