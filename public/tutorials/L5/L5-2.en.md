# Code Review Essentials

## AI-Generated Code Also Needs Review

Even AI-generated code requires manual review. Here are the core points for code review.

## Review Checklist

### 1. Security Issues

| Issue | Check Point |
|-------|-------------|
| SQL Injection | Whether parameterized queries are used |
| XSS | Whether output is escaped |
| Sensitive Info | Whether keys, passwords are exposed |
| Permissions | Whether privilege escalation risks exist |

### 2. Performance Issues

| Issue | Check Point |
|-------|-------------|
| N+1 Queries | Whether there are loop queries |
| Memory Leaks | Whether resources are unreleased |
| Large Data | Whether pagination is handled |
| Caching | Whether caching strategy exists |

### 3. Logic Issues

| Issue | Check Point |
|-------|-------------|
| Boundary Conditions | Whether null values, extreme values are handled |
| Error Handling | Whether try-catch exists |
| Business Logic | Whether it meets requirements |
| Dependencies | Whether dependency failures are handled |

### 4. Code Quality

| Issue | Check Point |
|-------|-------------|
| Naming | Whether variable/function names are clear |
| Comments | Whether key logic has comments |
| Complexity | Whether functions are too long |
| Duplication | Whether there is duplicate code |
