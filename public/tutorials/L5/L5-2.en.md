## 2. Phase 1: Requirements Analysis

### 2.1 Identify Target Users and Scenarios

**Target Users:**
- Junior developers: Learn code standards, improve code quality
- Mid-level developers: Quickly discover potential issues
- Tech leads: Control team code quality

**Usage Scenarios:**

| Scenario | Frequency | Pain Point |
|----------|-----------|------------|
| Self-review before code submission | Every commit | Easy to miss issues |
| Code Review | Every PR | Low review efficiency |
| Historical code audit | Periodic | Large code volume, difficult to check comprehensively |
| New employee training | Ongoing | Need to explain standards repeatedly |

**Security Rule Example (sql-injection.md):**

```markdown
# SQL Injection Prevention Rules

## Risk Level: 🔴 Critical

## Description
SQL injection is one of the most common security vulnerabilities. Attackers can execute arbitrary SQL statements by constructing malicious input.

## Detection Patterns

### Dangerous Patterns (Need to flag)
1. String concatenated SQL
   ```javascript
   // ❌ Dangerous
   const query = `SELECT * FROM users WHERE id = ${userId}`;
   ```

2. Non-parameterized dynamic queries
   ```javascript
   // ❌ Dangerous
   db.query("SELECT * FROM users WHERE name = '" + name + "'");
   ```

### Safe Patterns (Comply with standards)
1. Use parameterized queries
   ```javascript
   // ✅ Safe
   const query = 'SELECT * FROM users WHERE id = ?';
   db.query(query, [userId]);
   ```

2. Use ORM
   ```javascript
   // ✅ Safe
   const user = await User.findByPk(userId);
   ```

## Fix Suggestions
1. Always use parameterized queries or prepared statements
2. Use ORM frameworks (Sequelize, TypeORM, Prisma)
3. Input validation and escaping
4. Principle of least privilege (database account permissions)

## References
- [OWASP SQL Injection](https://owasp.org/www-community/attacks/SQL_Injection)
- [CWE-89](https://cwe.mitre.org/data/definitions/89.html)
```

### 3.4 Interface Definition

```typescript
// Skill interface definition
interface CodeReviewSkill {
  // Review a single file
  reviewFile(filePath: string, options?: ReviewOptions): Promise<ReviewResult>;
  
  // Review code snippet
  reviewCode(code: string, language: string, options?: ReviewOptions): Promise<ReviewResult>;
  
  // Review PR
  reviewPR(prData: PRData, options?: ReviewOptions): Promise<ReviewResult>;
  
  // Configure rules
  configureRules(rules: RuleConfig[]): void;
  
  // Get supported rules
  getSupportedRules(): Rule[];
}

// Review options
interface ReviewOptions {
  language?: string;           // Programming language
  severity?: SeverityLevel[];  // Severity levels to check
  categories?: Category[];     // Categories to check
  ignorePatterns?: string[];   // File patterns to ignore
  maxIssues?: number;          // Maximum number of issues to return
}

// Review result
interface ReviewResult {
  summary: {
    totalFiles: number;
    totalIssues: number;
    severity: {
      critical: number;
      high: number;
      medium: number;
      low: number;
    };
    categories: Record<Category, number>;
  };
  issues: Issue[];
  suggestions: Suggestion[];
  report: string;  // Markdown format report
}

// Issue definition
interface Issue {
  id: string;
  severity: SeverityLevel;
  category: Category;
  title: string;
  description: string;
  location: {
    file: string;
    line: number;
    column: number;
  };
  codeSnippet: string;
  fix?: {
    description: string;
    code: string;
  };
  references: string[];
}

// src/index.ts
import { SkillEngine } from './core/SkillEngine';
import { CodeParser } from './tools/CodeParser';
import { StaticAnalyzer } from './tools/StaticAnalyzer';
import { AIAnalyzer } from './tools/AIAnalyzer';
import { ReportGenerator } from './tools/ReportGenerator';
import { KnowledgeBase } from './knowledge/KnowledgeBase';

export class CodeReviewAssistant {
  private engine: SkillEngine;
  private knowledgeBase: KnowledgeBase;
  
  constructor(config: SkillConfig) {
    // Initialize knowledge base
    this.knowledgeBase = new KnowledgeBase(config.knowledge_base);
    
    // Initialize tools
    const tools = {
      parser: new CodeParser(config.tools.ast_parser),
      staticAnalyzer: new StaticAnalyzer(config.tools.static_analyzer),
      aiAnalyzer: new AIAnalyzer({
        knowledgeBase: this.knowledgeBase,
        promptTemplate: config.prompt_templates.analysis
      }),
      reportGenerator: new ReportGenerator(config.tools.report_generator)
    };
    
    // Initialize engine
    this.engine = new SkillEngine({
      workflow: config.workflow,
      tools: tools
    });
  }
  
  async reviewFile(filePath: string, options?: ReviewOptions): Promise<ReviewResult> {
    const context = {
      type: 'file',
      path: filePath,
      options
    };
    
    return this.engine.execute('standard_review', context);
  }
  
  async reviewCode(code: string, language: string, options?: ReviewOptions): Promise<ReviewResult> {
    const context = {
      type: 'code',
      code,
      language,
      options
    };
    
    return this.engine.execute('standard_review', context);
  }
  
  async reviewPR(prData: PRData, options?: ReviewOptions): Promise<ReviewResult> {
    const context = {
      type: 'pr',
      prData,
      options
    };
    
    return this.engine.execute('deep_review', context);
  }
}

**AI Analyzer Implementation:**

```typescript
// src/tools/AIAnalyzer.ts
import { OpenAI } from 'openai';

export class AIAnalyzer {
  private openai: OpenAI;
  private knowledgeBase: KnowledgeBase;
  private promptTemplate: string;
  
  constructor(config: AIAnalyzerConfig) {
    this.openai = new OpenAI({ apiKey: config.apiKey });
    this.knowledgeBase = config.knowledgeBase;
    this.promptTemplate = config.promptTemplate;
  }
  
  async analyze(
    code: string, 
    language: string, 
    context: AnalysisContext,
    staticIssues: StaticIssue[]
  ): Promise<AIssue[]> {
    // Get relevant rules from knowledge base
    const relevantRules = await this.knowledgeBase.getRelevantRules(code, language);
    
    // Build prompt
    const prompt = this.buildPrompt({
      code,
      language,
      context,
      staticIssues,
      rules: relevantRules
    });
    
    // Call AI for analysis
    const response = await this.openai.chat.completions.create({
      model: 'gpt-4',
      messages: [
        { role: 'system', content: 'You are a professional code review expert.' },
        { role: 'user', content: prompt }
      ],
      temperature: 0.3,
      max_tokens: 4000
    });
    
    // Parse AI output
    const analysis = response.choices[0].message.content;
    return this.parseAnalysis(analysis);
  }
  
  private buildPrompt(params: PromptParams): string {
    return this.promptTemplate
      .replace('{language}', params.language)
      .replace('{code}', params.code)
      .replace('{context}', JSON.stringify(params.context))
      .replace('{static_issues}', JSON.stringify(params.staticIssues))
      .replace('{rules}', params.rules.join('\n'));
  }
  
  private parseAnalysis(analysis: string): AIssue[] {
    // Parse AI output to extract structured issues
    const issues: AIssue[] = [];
    
    // Use regex to extract issue blocks
    const issueBlocks = analysis.match(/### \[.+?\][\s\S]+?(?=### |$)/g) || [];
    
    for (const block of issueBlocks) {
      issues.push(this.parseIssueBlock(block));
    }
    
    return issues;
  }
  
  private parseIssueBlock(block: string): AIssue {
    // Parse single issue block
    const severityMatch = block.match(/\[(\w+)\]/);
    const titleMatch = block.match(/-\s*([^\n]+)/);
    
    return {
      severity: severityMatch?.[1] as SeverityLevel || 'medium',
      title: titleMatch?.[1] || 'Unknown Issue',
      // ... other field parsing
    };
  }
}
```

### 4.3 Tool Integration

// .git/hooks/pre-commit
#!/bin/sh
# Auto-review before code commit

# Get staged files
STAGED_FILES=$(git diff --cached --name-only --diff-filter=ACM | grep -E '\.(js|ts|jsx|tsx|py)$')

if [ -z "$STAGED_FILES" ]; then
  exit 0
fi

# Call skill to review

if [ $? -ne 0 ]; then
  echo "❌ Code review failed. Please fix issues before resubmitting"
  exit 1
fi

echo "✅ Code review passed"
exit 0

## 6. Chapter Summary

### Core Process

Four phases of skills development:
1. **Requirements Analysis**: Define target users, usage scenarios, and functional requirements
2. **Skill Design**: Design architecture, knowledge base, workflow, and interfaces
3. **Development Implementation**: Write configuration, implement core logic, integrate tools
4. **Testing & Optimization**: Unit tests, integration tests, effectiveness evaluation, and continuous optimization

### Key Points

- **Knowledge Base is Core**: High-quality knowledge base determines the professionalism of the skill
- **Workflow should be Clear**: Clear steps and input/output ensure predictability
- **Tool Integration should be Flexible**: Support various usage scenarios (CLI, CI/CD, IDE)
- **Continuous Optimization is Key**: Continuously improve skill effectiveness through feedback
